Object

Reason のほとんどの時間、レコードを使って名前付きの値をグループ化するでしょう。 しかし、あるニッチな状況では、オブジェクトと呼ばれるこれに似た機能を使用したいと思うかもしれません。それらは幾分か柔軟で、異なるトレードオフが備わっています。

もし JavaScript から来るなら、大抵、おそらくプレーンな Reason オブジェクトを探してはいません。下部のヒントとコツのセクションにスキップしてください。

使い方

型宣言

オブジェクトは何かを保有することができるにも関わらず、型宣言を必要としません。それは . を除けばレコードのように見えます。

type tesla = {
  .
  color: string
};

始めにあるドットはこれが「閉じられた」オブジェクト型であることを表します。それはこの型に基づくオブジェクトは、正確にこの形状でなければならないこと意味します。

type car('a) = {
  ..
  color: string
} as 'a;

エリジオン（elision）とも呼ばれるふたつのドットは、これが「開かれた」オブジェクト型で、他の値やメソッドを含めることもできることを表しています。 開かれたオブジェクトは、多相でもあり、パラメータが必要です。

作成

type tesla = {.
  drive: int => int
};

let obj: tesla = {
  val hasEnvy = ref(false);
  pub drive = (speed) => {
    this#enableEnvy(true);
    speed
  };
  pri enableEnvy = (envy) => hasEnvy := envy
};

このオブジェクトはオブジェクト型が tesla で、パブリックメソッド drive を持ちます。それはオブジェクト内からのみアクセスできるプライベートメソッド enableEnvy も含みます。

ご覧のとおり、Reason オブジェクトは this にアクセスすることもできます。JavaScript オブジェクトの this の振る舞いは奇妙なことがあります。Reason の this は常に正確にオブジェクト自身を指します。

次の例は型をパラメータとして使用する開かれたオブジェクトを示しています。オブジェクト型パラメータは開かれたオブジェクト型のすべてのメソッドを実装するために必要です。

type tesla('a) = {
  ..
  drive: int => int
} as 'a;

let obj: tesla({. drive: int => int, doYouWant: unit => bool}) = {
  val hasEnvy = ref(false);
  pub drive = (speed) => {
    this#enableEnvy(true);
    speed
  };
  pub doYouWant = () => hasEnvy^;
  pri enableEnvy = (envy) => hasEnvy := envy
};

上記のオブジェクトを下記のように使用できます。

obj#doYouWant();

ヒントとコツ

If you come from JavaScript, a Reason object doesn't compile to a JS object. For that, try BuckleScript's special feature.
