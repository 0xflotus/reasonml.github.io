Let Binding

「let 束縛」は他の言語では「変数宣言/代入」と呼ばれることがあります。let は値に名前を与えます。それらは後に続くコードで見たり参照できます。

let greeting = "hello!";
let score = 10;
let newScore = 10 + score;
...

ブロックスコープ

束縛は {} 内で有効です。

if (displayGreeting) {
  let message = "Enjoying the docs so far?";
  print_endline(message)
};
/* `message` not accessible here! */

束縛は不変

「変わらない」の「不変」です。 一度値に束縛したら、それは他の値を参照することはできません。（明示的に変更可能な値と明示した場合を除いて。後で説明します）。 しかし、以前の束縛をシャドウして、同じ名前で新しい束縛を作ることもできます。

let message = "hello";
print_endline(message); /* Prints "hello" */
let message = "bye";
print_endline(message); /* Prints "bye" */

ヒントとコツ

束縛は {} 内で有効なので、周辺に匿名のスコープを作ることができます:

let message = {
  let part1 = "hello";
  let part2 = "world";
  part1 ++ " " ++ part2
};
/* `part1` and `part2` not accessible here! */

これらの行の後で束縛が濫用されることを防ぎます。

設計方針

Reason は OCaml で裏付けされています。let 束縛は OCaml のシンタックスでは次のようになります:

let a = 1 in
let b = 2 in
a + b

これは概念的には下記の形式に読み替えられます:

let a = 1 in
  let b = 2 in
    a + b

Reason では次のとおりです:

let a = 1;
let b = 2;
a + b;

これを思い起こさせるかもしれません:

/* Reason syntax */
(a) =>
  (b) =>
    a + b;

これらは厳密には同じではありませんが、let が単なる式で関数みたいなものと感じられたら幸いです！ Reason では、in を視覚的に馴染み深い ; に変えましたが、根底にある式の優雅さは変わっていません。
