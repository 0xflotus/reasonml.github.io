Integer & Float

整数

32ビット、必要に応じて切り捨てられます。Reason はそれらに一般的な操作を提供します: +, -, *, /, など。

使い方

標準ライブラリの int32 モジュールを参照してください。JS のコンパイルについては、Js.Int を参照してください。

ヒントとコツ

JavaScript の数値にバインドする時は気をつけてください！長いものは切り捨てられます。代わりに浮動小数点として JS の数値をバインドします。

浮動小数点数

浮動小数点数では他の演算子を使う必要があります: +., -., *., /., など。0.5 +. 0.6 のような。

使用法

現在の標準ライブラリには Float モジュールはありません。JS のコンパイルについては、Js.Float を参照してください。

設計方針

「なぜ私は int と float の両方にオーバーロードされた + を使うことができないのですか？ なぜ私は素晴らしい型と相互運用性とコミュニティがあって高性能な高い言語を見つけるたびに、この種の欠点が見つかるのですか？」

そこです、そこ。 現在の型システムでは、多相演算子はコンパイラにハードコードする必要があります。 物理的な多相な比較演算子である == は、そのような演算子です。+ とその他諸々はそうではありません。 希望通りに機能させるための改善は進行中です。 それまでの間は、とりあえずそのまま進めておきましょう =)。

加えて、むしろ浮動小数点数は Reason/OCaml ネイティブでは特殊です。 非常に興味深い最適化を学ぶことに興味があれば！ここをチェックしてください。
