List & Array

リスト

リストは:

  • 一様
  • イミュータブル
  • 先頭に要素を追加するのが速い
let myList = [1, 2, 3];

Reason のリストはシンプルで、単方向リストです。

使い方

サイズ変更が可能で、prepend （先頭に追加する）、分割、が高速で、なおかつそれらすべてがイミュータブルでさらに効率的なリストを使うでしょう！

標準ライブラリは List モジュール (およびラベル付き引数版の ListLabels）を提供しています。

不変リストの前置

ちょうど List.cons に相当する、スプレッドシンタックスを使用します。

let myList: list(int) = [1, 2, 3];
let anotherList = [0, ...myList];

myList は変更されていません。 anotherList は、現在 [0, 1, 2, 3] です。 これは効率的です。（線形ではなく定数時間） anotherList の最後の三要素は、myList と共有されます！

注意 [a, ...b, ...c] はシンタックスエラーになります。 リストの複数のスプレッドはサポートしていません。 これは b のそれぞれの要素が c の先頭に一つづつ追加されるので、偶発的な線形演算（O(b)）になります。 これのために List.concat が使用できます。

パフォーマンスと割当てのオーバーヘッドが線形（O(n)）になるため、list の真ん中にある任意の要素を更新することはお薦めしません。

アクセス

switch（パターンマッチングのセクションに出てきます）は、通常、リストの要素にアクセスする時に使います。

let message =
  switch (myList) {
  | [] => "This list is empty"
  | [a, ...rest] => "The head of the list is the string " ++ string_of_int(a)
  };

任意の要素にアクセスするには List.nth を使用します。

ヒントとコツ

空リストは何も気にせず自由に使ってください。 ヴァリアントのリストに関するセクション で説明した通り、空リストは実際にはパラメータなしのヴァリアントのコンストラクタで、単なる整数にコンパイルされます。 余分なメモリ割り当てはありません。

Prepend がどのようにして不変で同時に O(1) であるかについて理解するには、前述のリンクを参照してください。

設計方針

将来的には、不変でサイズ変更が可能で、オールラウンドの高速操作を特徴とする、すぐに使えるリストデータ構造を提供するかもしれません。

配列

配列はリストに似ていますが、下記の点が異なります。

  • ミュータブル
  • ランダムアクセス・更新が速い
  • ネイティブで固定長（JavaScript では可変長）

[| と |] で囲みます。

let myArray = [|"hello", "world", "how are you"|];

使用法

標準ライブラリの Array と ArrayLabel モジュール。 JS のコンパイルについては、おなじみの Js.Array API もあります。

配列の要素へのアクセスと更新は下記のようにします。

let myArray: array(string) = [|"hello", "world", "how are you"|];

let firstItem = myArray[0]; /* "hello" */

myArray[0] = "hey";

/* now [|"hey", "world", "how are you"|] */

上記の例の配列のアクセス/更新は Array.get/Array.set の単なるシンタックスシュガーです。

ヒントとコツ

JavaScript にコンパイルすると、Reason の配列は、そのまま JavaScript の配列にマッピングされること、逆もそうであることがわかります。 したがって、たとえ配列がネイティブでは固定長でも、Js.Array API でリサイズすることができます。 ご心配なく。
