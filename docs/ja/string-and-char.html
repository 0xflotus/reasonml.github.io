String & Char

文字列

Reason の文字列はダブルクォートで区切ります（シングルクォートは以下の文字型のために予約されています）。

let greeting = "Hello world!";
let multilineGreeting = "Hello
 world!";

文字列内の特殊文字はエスケープする必要があります：

let oneSlash = "\\";

文字列を結合するには ++を使います：

let greetings = "Hello " ++ "world!";

クォートされた文字列

文字列のための特別な構文があります。

  • 前述のような複数行文字列
  • 特殊文字がエスケープされない
  • 特別なプリプロセッサ用のフック
let greetingAndOneSlash = {|Hello
World
\
Hehe...
|};

特殊文字をエスケープする必要がないことと、組み込みの変数補完をしないことを除けば、JavaScript のバックティック文字列補完に似ています。 ただ、後者の機能は BuckleScript のように簡単に実現できます：

let world = {js|世界|js}; /* Unicode 文字をサポート */
let helloWorld = {j|你好，$world|j}; /* Unicode と変数展開をサポート */

BuckeScript の特別なプリプロセッサは、文字列の周辺にある js や j のようなマーカーを探して、別のものに変換することができます。

使い方

多くの文字列操作は標準ライブラリにあります。 JS のコンパイルについては、 BuckleScript API ドキュメント の Js.String API あたりを参照してください。 Reason の文字列は JavaScript の文字列にマッピングされるので、両方の標準ライブラリでミックスしたりマッチさせたりできます。

ヒントとコツ

https://twitter.com/jusrin00/status/875238742621028355

表現力豊かなタイプシステムがあります！型付けされていない言語では、しばしば、次のようにして文字列の意味をオーバーロードすることでしょう：

  • 一意の id：var BLUE_COLOR = "blue"
  • データ構造の中に識別子を入れる：var BLUE = "blue"; var RED = "red"; var colors = [BLUE, RED]
  • オブジェクトのフィールド名：person["age"] = 24
  • 列挙型：if (audio.canPlayType() === 'probably') {...} (ಠ_ಠ)
  • Reason の代替を知った後には、すぐに恐ろしいとわかるであろう、その他のクレイジーなパターン。

貧弱な文字列型をオーバーロードすればするほど、型システム（またはチームメイト）はあなたを助けられなくなります！ Reason は、上記のユースケース（例えば後のセクションのヴァリアント）の代わりとなる、簡潔で高速なメンテナブルな型とデータ構造を提供します。

ネイティブコンパイルでは、 Reason の文字列は直接パフォーマンスを分析できる単純な表現にコンパイルされ、手動のパフォーマンスチューニングが必要になることもあります。 例えば、"hi " ++ "how " ++ "are " ++ "you?" のような素朴な文字列を連結すると、中間の文字列 "are you?" と "how are you?" が不必要に割り当てられます。 この場合、String.concat が適しています。 ある意味では、学校で学んだ伝統的なランタイム分析が最終的にはもう一度役立つかもしれません。

Under JavaScript compilation, a Reason string maps to a JavaScript string and vice-versa, so no such above concern or analysis opportunities apply.

設計方針

クォートされた文字列の特殊文字をエスケープしないという特徴は、正規表現のような DSL をスッキリ記述することを可能にします：

let r = Str.regexp({|hello \([A-Za-z]+\)|});

下記とは対照的に

let r = Str.regexp("hello \\([A-Za-z]+\\)");

JS コンパイルでは、Str は利用できないので、代わりに [%bs.re] と Js.Re を使用します。

Reason/OCaml は、単純なネイティブの文字列の実装を通して、ここで賢明さよりもシンプルさを重視しています。 あまりに洗練された文字列の実装は、時に裏目に出ることがあります。

文字

Reason は 1 文字の文字列の型を持ちます：

let firstLetterOfAlphabet = 'a';

備考：文字は Unicode または UTF-8 をサポートしていません。

ヒントとコツ

余分な速度のために、文字は 0 から 255 の範囲の整数にコンパイルされます。 また、パターンマッチ（後で説明します）することもできます：

let isVowel = (theChar) =>
  switch (theChar) {
  | 'a'| 'e'| 'i'| 'o'| 'u'| 'y' => true
  | _ => false
  };

ある文字列をある文字に変換するには、"a".[0] を使います。ある文字をある文字列に変換するには、String.make(1, 'a') を使います。
