Extra Goodies

ブラウザ拡張: Reason-tools

Reason-tools を使用すると、どちらのシンタックスで記述されたチュートリアルやドキュメントを参照しているときにも、OCaml のシンタックスと Reason のシンタックスを簡単に切り替えることができます。

BuckleScript

BuckleScript は、Reason の JS コンパイルで使用するエンジンなので、このドキュメンテーションの残りの部分で度々目にすることになります。 そのグローバルバイナリは、npm install -g bs-platform でインストールされます。

その他のユーティリティ

以前のセクションで実施したグローバルインストールで、ここで紹介するいくつかの便利なツールがインストールされています。

Refmt

refmt ("Reason format") は、エディタの自動整形機能を強化するバイナリです。 あなたのコードを取り込み、上手く整形します。 Reason と OCaml のシンタックスを相互に変換する役割も果たします。 Reason コミュニティでは、さまざまなプロジェクトで一貫したコードスタイルを実現するために、また、時間の無駄になる手作業によるフォーマッティング & スタイルに関する議論を避けるために、このツールを使用しています。

It responsively formats the code based on the editor width. In other words, it doesn't just naively break to the next line at a certain characters limit; it solves the layout constraints and arranges your code accordingly.

refmt はターミナルで直接利用することもできます。 例えば、エディタ外でコードをフォーマットするには、refmt --in-place myFile.re を実行します。 すべてのオプションについては refmt --help を参照してください。

Merlin

Merlin は型ヒント、リファクタリング、リアルタイムエラー、定義ジャンプ、その他もろもろを実現する根幹となるエンジンです。 （私たちのエディタに） 手動で呼び出すことはありませんが、そのコマンドラインの名前は ocamlmerlin と呼ばれます。（エディタがそれ自身を起動して問い合わせます）

Merlin をあなたのプロジェクトを理解するように設定するには、ルートに .merlin ファイルを作成します（ドキュメントはここ）。 JS のワークフローでは、この設定は BuckleScript の bsb によって自動的に生成されます。

REPL

Reason には rtop と呼ばれる REPL が付属しています。これを呼び出すと、インタラクティブにコードを評価できます。インテリジェントで型駆動のオートコンプリートを備えています。

#quit; を使用して REPL セッションを閉じます。

注意 rtop は、現在 packages と externals が簡単には動作しません。Try プレイグラウンドでコードを実行してみることをおすすめします。

re:bench

re:bench はオンラインベンチマークプレイグラウンドです。 Reason、OCaml そして JavaScript で書かれたテストケースをサポートし、一意な URL でベンチマークを共有することができます。 BuckleScript を使って JS にコンパイルされます。

Redux

Redex: Reason パッケージレジストリ。

ocamlc, ocamlopt, ocamlrun, rebuild

ocamlc と ocamlopt は素の ocaml コンパイラです。
