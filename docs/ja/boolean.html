Boolean

ブール値は bool 型で、true または false のいずれかになります。一般的な操作：

  • &&: 論理積
  • ||: 論理和
  • !: 論理否定
  • <=, >=, <, >
  • ==: 物理的に等しい、データ構造を深く比較する: (1, 2) == (1, 2) は true。便利ですが注意して使用してください。
  • ===: 参照が等しい、浅く比較する。 (1, 2) === (1, 2) は false。let myTuple = (1, 2); myTuple === myTuple は true。
  • !=: 物理的に等しくない
  • !==: 参照が等しくない

使い方

備考: BuckleScript は JavaScript の true と false のバインディングを提供します。それは Reason/OCaml の true と false と同じものではありません！ 適切な変換（Js.to_bool と `Js.Boolean.to_js_boolean）をせずに区別なくそれらを使用しないでください。

ヒントとコツ

趣味よく物理的な比較を使用する。 これは便利ですが、誤って2つの深くネストされたデータ構造を比較して、大きなパフォーマンス低下を引き起こす可能性があります。 何が「等しい」とみなされるかは必ずしも明らかではありません。 例えば、foo` というデータの一部は、遅延された `foo` に等しいでしょうか？ 理想的には、それはプラガブルとしたいところです。 今後の変更で、これを可能にし、信頼できるものにします。もし興味があれば、[modular implicit](https://www.reddit.com/r/ocaml/comments/2vyk10/modular_implicits/) を確認してみてください。

設計方針

このセクションはヴァリアントに関する知識を前提としています。もし、あなたがこのガイドを読むのが初めてならば、気軽に後でこのページに戻って来てください！

ブール値は（概念的に）ヴァリアントの特殊ケースです: type bool = True | False。 設計上、型システムのブール型をハードコードする必要性を華麗に取り除きます。 欠点は、ヴァリアントのように、コンストラクタが高速だが読みにくくコンパイルされる点です。 これは、BuckeScript が、後の段階で、Reason の true/false を JavaScript の true/false にコンパイルするための情報を欠いているためです。
