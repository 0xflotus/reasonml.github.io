Destructuring

構造化（または分割）代入 ("Destructuring") は、データ構造からフィールドを抽出する視覚的に簡潔な方法です。通常、変数を使用するところならば、どこでも構造化代入を使用できます。

使い方

以下は変数を束縛します: ten = 10, twenty = 20

let someInts = (10, 20);
let (ten, twenty) = someInts;

以下は変数を束縛します: name = "Guy", age = 30

type person = {name: string, age: int};
let somePerson = {name: "Guy", age: 30};
let {name, age} = somePerson;

フィールドを取り出す時に任意でリネームすることができます。以下は、これらの代わりです: n = "Guy", a = 30

let {name: n, age: a} = somePerson;

構造化代入でも型注釈を指定できます。

let (ten: int, twenty: int) = someInts;
let {name: (n: string), age: (a: int)} = somePerson;

Destructuring a function's labeled arguments is also possible.

type person = {
  name: string,
  age: int
};

let someFunction = (~person as {name}) => {
  /* ここでは `name` が使えます */
};

let otherFunction = (~person as {name} as thePerson) => {
  /* ここでは `name` と、レコード全体を表す `thePerson` の両方が使えます */
};

パターンマッチングのセクションを読み進めてください。構造化代入のよりクレイジーな例があります。

ヒントとコツ

構造化代入を使うと、中間変数に名前を付ける必要がなくなり、あなたのコードをもっと簡潔にすることができます。使ってください！ただし、乱用して、コードを過度に入れ子にすることは避けてださい。

もし、定義が現在のファイルにないレコードまたはヴァリアントを構造化代入する場合は、明示的に注釈を付ける必要があります。 こことここを参照してください。
