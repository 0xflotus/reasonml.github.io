Booléen

Un booléen possède le type bool et peut valoir true ou false. Les opérations les plus communes sont :

  • &&: ET logique
  • ||: OU logique
  • !: NON logique.
  • <=, >=, <, >
  • ==: égalité structurelle, compare les structures de données profondément : (1, 2) == (1, 2) vaut true. Pratique, mais à utiliser avec prudence
  • ===: égalité structurelle, compare superficiellement. (1, 2) === (1, 2)vaut false. let myTuple = (1, 2); myTuple === myTuple vaut true.
  • != : inégalité structurelle
  • !== : inégalité référentielle

Utilisation

Note: Les true and false Reason/Bucklescript compilent en true and false JavaScript depuis Bucklescript 3.0. Avant cette version, vous deviez utiliser Js.to_bool et Js.Boolean.to_js_boolean mais heureusement ce n'est plus le cas !

Conseils & astuces

Utilisez l'égalité structurelle à bon escient. Elle est pratique, mais peut accidentellement vous faire comparer deux structures de données profondément imbriquées et ainsi entraîner une baisse de performance. Il n'est pas toujours évident de discerner ce qui est considéré comme étant "égal". Par exemple, est-ce qu'une donnée foo est égale à un foo lazy ? Idéalement, cela aurait été pluggable. Des futurs changements sont prévus pour rendre cela possible et fiable. Si vous êtes intéressé(e), jetez un coup d'oeil à "modular implicit“.
