Binding let

Dans un autre language, un let binding serait sûrement appelé « déclaration/affectation de variable ». let donne des noms à une valeur. Ils peuvent être vus et référencés par le code qui suit le let binding.

let greeting = "hello!";
let score = 10;
let newScore = 10 + score;
...

Portée de bloc

On peut limiter la portée d'une variable avec {}.

if (afficherSalutation) {
  let message = "Salut à vous, lecteurs !";
  print_endline(message)
};
/* `message` n'est pas accessible ici ! */

Les bindings sont immutables

"Immutables" comme dans "ne changent pas". Une fois qu'un binding se réfère à une valeur, il ne peut se référer à rien d'autre (sauf s'il contient explicitement une valeur mutable, point abordé plus loin). Cependant, vous pouvez créer un nouveau binding du même nom qui serait une copie superficielle du binding précédent. À partir de ce moment-là, le binding se référera à la valeur nouvellement attribuée.

let message = "hello";
print_endline(message); /* Affiche "hello" */
let message = "bye";
print_endline(message); /* Affiche "bye" */

Trucs & astuces

Puisque la portée des variables est limitée aux {}, il est possible de créer un bloc anonyme autour d'elles :

let message = {
  let part1 = "hello";
  let part2 = "world";
  part1 ++ " " ++ part2
};
/* `part1` and `part2` ne sont pas accessibles ici ! */

Cela empêche les variables d'être mal utilisées après ces lignes.

Choix de conception

Reason repose sur OCaml pour fonctionner. Voici à quoi ressemble un let binding en OCaml :

let a = 1 in
let b = 2 in
a + b

Ce qui pourrait conceptuellement être lu sous la forme suivante :

let a = 1 in
  let b = 2 in
    a + b

Ce qui correspond à ceci en Reason :

let a = 1;
let b = 2;
a + b;

Ce qui peut potentiellement vous rappeler :

/* syntaxe Reason */
(a) =>
  (b) =>
    a + b;

Même si ce n'est pas exactement la même chose, on se rend compte que let est juste une expression, semblable à une fonction! En Reason, nous avons changé in en ; parce que cela est plus familier visuellement; mais ne laissez pas cela cacher l'élégance sous-jacente des expressions.
