Type!

Les types sont la fonctionnalité phare de Reason ! Ici nous essayerons de vous donner un aperçu sur pourquoi cela fait rêver tout le monde.

Cette section introduit brièvement la syntaxe des types pour que vous puissiez lire les sections suivantes sans vous perdre. Des détails plus avancés sur les types peuvent être trouvés dans la section Plus sur les types.

Annotations

Le let binding suivant ne contient aucun type écrit :

let score = 10;

Reason sait que score est un int, simplement en regardant 10. Cela s'appelle l'inférence de type.

Mais vous pouvez aussi faire le choix d'écrire explicitement les types :

let score: int = 10;

Vous pouvez aussi annoter chaque expression en l'entourant de parenthèses :

let myInt = 5;
let myInt: int = 5;
let myInt = (5: int) + (4: int);
let add = (x: int, y: int) : int => x + y;
let drawCircle = (~radius as r: int) : unit => ...;

Note : dans la dernière ligne, (~radius as r: int) est un argument étiqueté. Plus à ce sujet ici.

Alias

Vous pouvez créer des alias pour les noms des types. Ils seront équivalents :

type scoreType = int;
let x: scoreType = 10;

Choix de conception

Reason s'appuie sur OCaml pour son fonctionnement, dont le système de type est le produit de dizaines d'années de travail. Voici quelques points clés :

  • Les types peuvent être inférés. Le système de type déduit tout seul les types, mêmes si vous ne les écrivez pas vous-même. Cela accélère grandement la phase de prototypage. De plus, certaines fonctionnalités d'éditeurs de texte comme codelens de VSCode vous montrent tous les types pendant que vous écrivez du code.

  • La couverture en type est toujours de 100%. Aucun outil de « couverture de type » n'est nécessaire ! Chaque morceau de code en Reason a un type.

  • Le système de type est « juste ». Cela signifie que, du moment que votre code compile, chaque type garantit d'être juste. D'habitude, dans les autres systèmes de type, le fait qu'un type affirme qu'il est, par exemple, « un entier qui n'est jamais nul » ne signifie pas qu'il n'est véritablement jamais nul. En revanche, un programme en Reason pur n'a aucun bug de nullité.
