Démarrage rapide

npm install -g bs-platform  # installe le compilateur BuckleScript
bsb -init my-first-app -theme basic-reason  # crée le projet my-first-app/
cd my-first-app
npm run build  # compile dans src/Demo.bs.js en utilisant le compilateur BuckleScript
node src/Demo.bs.js  # lance la démo via NodeJS

Vous devriez voir un "hello world" s'afficher :

> node src/Demo.bs.js
Hello, BuckleScript and Reason!

Nous utilisons le compilateur BuckleScript pour compiler Reason en JavaScript, puis nous utilisons NodeJS pour exécuter le JavaScript. Après avoir exécuté le compilateur, ce qui reste suit le workflow JavaScript habituel (comme l'exécution de node ou le navigateur).

Lors du développement, au lieu d'exécuter npm run build à chaque fois, exécutez npm run start pour démarrer un watcher qui recompile lors du changement de fichier.

By default, the basic-reason theme configures BuckleScript to output compiled JavaScript files to the src directory, alongside the Reason files they were produced from. If you prefer to keep your handwritten and generated files separate, editing bsconfig.json to set in-source to false will cause the compiled files to be output to the lib directory instead.
