Ejemplos de novato

Un ejemplo vale mas que mil palabras.

Esta sección está dedicada a los recién llegados intentando averiguar expresiones generales y convenciones en Reason y BuckleScript. Si eres un principiante que tiene una buena idea que sirva como ejemplo, ¡por favor sugiere una edición!

Utilizando el tipo option

option es una variante que viene con la biblioteca estándar. Evita la necesidad de valores nulos en otros lenguajes.

let possiblyNullValue1 = None;
let possiblyNullValue2: option(string) = Some("Hola@");

switch (possiblyNullValue2) {
| None => print_endline("Nada que ver aquí.")
| Some(message) => print_endline(message)
};

Creando un tipo parametrizado

type universityStudent = {gpa: float};

type response('studentType) = {
  status: int,
  student: 'studentType
};

let result: response(universityStudent) = fetchDataFromServer();

Creando un objeto JS

Suponiendo que estás compilando a JS, por supuesto.

let obj1 = {
  "name": "John",
  "age": 30
};
/* Compila a un objeto JS que es exactamente igual al que tu estas viendo */

Ten en cuenta que lo anterior no es un registro; las claves se representan como cadenas. Eso es endulzamiento sintáctico (syntactic sugar) de Reason para bs.obj. El tipo es inferido. El siguiente ejemplo explícitamente anota los tipos.

Anotando los tipos de un objeto JS

type payload = {.
  "nombre": string,
  "edad": int
};

let obj1: payload = {"nombre": "John", "edad": 30};

Ten en cuenta que {.nombre: string, edad: int} es la sintaxis de una declaración de tipo de objeto de Reason/OCaml (¡no un registro!). Es mejorado en Js.t para que BuckleScript vea el tipo completo y lo compile correctamente a un objeto de JavaScript regular. Los objetos ordinarios, no mejorados de OCaml, son compilados a algo diferente (actualmente rara vez se necesitan).

Vinculando Binding a un módulo JS con Export por defecto

Suponiendo que el módulo se llama store.js, y tiene una Export por defecto, además de un método llamado getDate.

type store = {. "getDate": [@bs.meth] (unit => float)};
[@bs.module] external store : store = "./store";

Js.log(store);
Js.log(store##getDate());

Comprobando si hay tipos JS que aceptan valores Null usando el tipo option

Para una función a cuyo argumento se pasa un valor de JavaScript que potencialmente es null o undefined, es idiomático convertirlo en un option de Reason. La conversión se realiza a través de las funciones de ayuda en el módulo Js.Nullable de Bucklescript. En este caso, to_opt:

let greetByName = (possiblyNullName) => {
  let optionName = Js.Nullable.to_opt(possiblyNullName);
  switch (optionName) {
  | None => "Hey"
  | Some(name) => "Hola " ++ name
  }
};

Esta comprobación compila a possiblyNullName == null en JS, así verifica la presencia de null o undefined.
