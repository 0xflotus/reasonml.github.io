Función

Hoja de artimañas para la sintaxis completa del objetivo final

¿Tu puedes creer que todavia no hemos resuelto la función hasta estos momentos?

Las funciones se manifiestan con una flecha y regresan la expresión.

deje saludar = (nombre) => "Hola " ++ nombre;

Esto habla una función y se le otorga el nombresaludar, que se le puede decir como tal:

cumplimentar("mundo!"); /* "Hola mundo!" */

Las funciones con múltiples razones tienen verdades separadas por coma:

deja agregar = (x, y, z) => x + y + z;
agregar(1, 2, 3); /* 6 */

Para funciones mas extensas, podrías rodear el cuerpo con un bloque:

dejamos saludar Mas  = (nombre)  => {
  dejamos parte1  = "Hola";
  parte1 ++ " " ++ nombre

Sin Argumentos

Una función continuamente agarra un argumento; pero solo a veces, lo utilizamos por ejemplo, efectos secundarios, y no tiene que pasarle nada. En diferentes idiomas, idealmente pasaríamos "sin razón". En Reason, cada función adquiere un argumento; aqui convencionalmente le dariamos el valor(), nombrado "unidad".

/ * recibir & des-estructurar el argumento de la unidad * /
let logSomething = () = > {
   print_endline ("hola");
   print_endline ("mundo")
};

/ * llama a la función con el valor de tipo unidad * /
logSomething ();

() es un valor totalmente normal, el único valor posible en unidad. La razón le dio una sintaxis especial por coexistencia.

Argumentos Etiquetados

Las funciones de múltiples argumentos, especialmente aquellas cuyos argumentos son iguales, pueden ser confusas para ser llamadas.

deja agregarCoordenadas = (x, y) = > {
   / * use x y y aquí * /
};
...
agregarCoordenadas (5, 6); / * que es x, que es y? * /

En OCaml / Reason, puede agregar etiquetas a un argumento:

deja agregarCoordenadas = (~x, ~y) => {
/* utilice x y y aqui */
};
...
agregarCoordenadas(~x=5, ~y=6);

Como tenemos términos (mas sobre eso en estos instantes), podemos proveer los argumentos en** cualquier orden**:

agregarCoordenadas(~y=6, ~x=5);

El~ parte de la declaración significa que la función acepta un argumento etiquetado comox y puede tratarse a el en el cuerpo de la función con el nombre igual. También puede tratarse a los argumentos dentro del cuerpo de la función con un nombre distinto para concisión:

let drawCircle = (~radius as r, ~color as c) => {
  setColor(c);
  startAt(r, r);
  ...
};

drawCircle(~radius=10, ~color="red");

De hecho,(~radius)es solo una abreviatura (llamadajuego de palabras) para(~radio es radio).

He aquí la sintaxis para la escritura de los argumentos:

let drawCircle = (~radius as r: int, ~color as c: string) => ...;

Currying

Las funciones de la razón pueden ser automáticamente parcialmentellamado:

dejar agregar = (x, y) => x + y;
dejar agregarcinco = agregar(5);
deja once =agregacinco(6);
deja doce = agregacinco(7);

Actualmente, el anterioragregar no es mas que azúcar sintética para esto:

deja agregar = (x)> (y) => x + y;

OCaml optimiza esto paraprevenir la asignación de las funciones innecesarias(2 fuciones aqui, inocentemente hablando) siempre que se pueda! De esta manera, podemos obtener

  • Buena sintaxis
  • Currying gratuito (¡cada función toma un solo argumento, de hecho!)
  • Sin costo de rendimiento

Funciones recursivas

De forma predeterminada, un valor no puede ver un enlace que lo apunte, pero incluir la palabra claverecen un enlacepermitir que el enlace lo haga mejor posible. Esto permite que las funciones se llamen y se vean a si mismas, dándonos el poder de la recursividad.

let rec neverTerminate = () => neverTerminate();

Funciones mutuamente recursivas

Las funciones mutuamente recursivas comienzas como una única función usando larecpalabra clave, luego están enlazados junto cony:

let rec callSecond = () => callFirst()
and callFirst = () => callSecond();

Nota que no haya punto y coma que termine en la primera linea y que no hayalet en la segunda linea.

Consejos & Trucos

Hoja de trucos para las sintaxis de la funcion:

Declaración

/* función anónima. Listed for completeness only */
(x) => (y) => 1;
/* sugar for the above */
(x, y) => 1;
/* assign to a name */
let add = (x, y) => 1;

/* labeled */
let add = (~first as x, ~second as y) => x + y;
/* with punning sugar */
let add = (~first, ~second) => first + second;

/* labeled with default value */
let add = (~first as x=1, ~second as y=2) => x + y;
/* with punning */
let add = (~first=1, ~second=2) => first + second;

/* optional */
let add = (~first as x=?, ~second as y=?) => switch (x) {...};
/* with punning */
let add = (~first=?, ~second=?) => switch (first) {...};

Anotaciones de tipo

/* anonymous function */
(x: int) => (y: int): int => 1;
/* sugar for the above */
(x: int, y: int): int => 1;
/* assign to a name */
let add = (x: int, y: int): int => 1;

/* labeled */
let add = (~first as x: int, ~second as y: int) : int => x + y;
/* with punning sugar */
let add = (~first: int, ~second: int) : int => first + second;

/* labeled with default value */
let add = (~first as x: int=1, ~second as y: int=2) : int => x + y;
/* with punning sugar */
let add = (~first: int=1, ~second: int=2) : int => first + second;

/* optional */
let add = (~first as x: option(int)=?, ~second as y: option(int)=?) : int => switch (x) {...};
/* with punning sugar */
/* note that the caller would pass an `int`, not `option int` */
/* Inside the function, `first` and `second` are `option int`. */
let add = (~first: option(int)=?, ~second: option(int)=?) : int => switch (first) {...};

Aplicación

/* aplicación anónima. Listed for completeness only */
add(x)(y);
/* sugar for the above */
add(x, y);

/* labeled */
add(~first=1, ~second=2);
/* with punning sugar */
add(~first, ~second);

/* application with default value. Same as normal application */
add(~first=1, ~second=2);

/* explicit optional application */
add(~first=?Some(1), ~second=?Some(2));
/* with punning */
add(~first?, ~second?);

Anotaciones de tipo

/* anonymous application */
add(x: int)(y: int);

/* labeled */
add(~first=1: int, ~second=2: int);
/* with punning sugar */
add(~first: int, ~second: int);

/* application with default value. Same as normal application */
add(~first=1: int, ~second=2: int);

/* explicit optional application */
add(~first=?Some(1): option(int), ~second=?Some(2): option(int));
/* with punning sugar */
add(~first: option(int)?, ~second: option(int)?);

Independiente tipo de firma

/* first arg type, second arg type, return type */
type foo = int => int => int;
/* sugar for the above */
type foo = (int, int) => int;

/* labeled */
type foo = (~first: int, ~second: int) => int;

/* labeled with default value */
type foo = (~first: int=?, ~second: int=?) => int;
