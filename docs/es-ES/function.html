Función

Hoja de referencia completa de la sintaxis de funciones al final

¿Te puedes creer que todavía no hemos cubierto las funciones?

Las funciones son declaradas con una flecha y devuelven la expresión.

let greet = (name) => "¡Hola " ++ name;

Esto declara una función y le asigna el nombre greet, al que puedes llamar así:

greet("mundo!"); /* "Hola mundo!" */

Las funciones multi argumento tienen los mismos separados por comas:

let add = (x, y, z) => x + y + z;
add(1, 2, 3); /* 6 */

Para funciones mas extensas, rodearías el cuerpo con un bloque:

let greetMore = (name) => {
  let part1 = "Hello";
  part1 ++ " " ++ name
};

Sin Argumento

Una función siempre recibe un argumento; pero a veces, la utilizamos por ejemplo para realizar efectos secundarios, y no hay nada que pasarle. En otros lenguajes, conceptualmente pasaríamos en "sin argumento". En Reason, cada función recibe un argumento; aquí convencionalmente pasaríamos el valor(), llamado "unit".

/* recibir & des-estructurar el argumento unit */
let logSomething = () => {
  print_endline("hola");
  print_endline("mundo")
};

/* llamar a la función con el valor de tipo unit */
logSomething();

() es un valor totalmente normal, el único valor posible en unit. Reason le dio una sintaxis especial por conveniencia.

Argumentos Etiquetados

Las funciones con múltiples argumentos, especialmente aquellas cuyos argumentos son del mismo tipo, pueden ser confusas al ser llamadas.

let addCoordinates = (x, y) => {
  /* usa x e y aquí */
};
...
addCoordinates(5, 6); /* ¿cual es x, cual is y? */

En OCaml/Reason, puedes adjuntar etiquetas a un argumento:

let addCoordinates = (~x, ~y) => {
  /* usa x e y aqui */
};
...
addCoordinates(~x=5, ~y=6);

Como tenemos currying (más sobre esto después), podemos pasar los argumentos en cualquier orden:

addCoordinates(~y=6, ~x=5);

The ~x part in the declaration means the function accepts an argument labeled x and can refer to it in the function body by the same name. También puedes referirte a los argumentos dentro de la función por un nombre diferente por brevedad:

let drawCircle = (~radius as r, ~color as c) => {
  setColor(c);
  startAt(r, r);
  ...
};

drawCircle(~radius=10, ~color="red");

De hecho, (~radius) es solo una abreviatura (called punning) for (~radius as radius).

Esta es la sintaxis para el tipado de argumentos:

let drawCircle = (~radius as r: int, ~color as c: string) => ...;

Currying

Las funciones en Reason pueden ser llamadas parcialmente de manera automática:

let add = (x, y) => x + y;
let addFive = add(5);
let eleven = addFive(6);
let twelve = addFive(7);

En realidad, el anterior add no es mas que un endulzamiento sintáctico syntactic sugar de esto:

let add = (x) => (y) => x + y;

OCaml optimiza esto para evitar la innecesaria asignación de la función (2 funciones aquí, ingenuamente hablando) ¡siempre que se pueda! Así, obtenemos

  • Sintaxis clara
  • Currying de forma gratuita (cada función recibe un solo argumento, ¡de hecho!)
  • Sin coste en el rendimiento

Argumentos etiquetados opcionales

Los argumentos de función etiquetados se pueden hacer opcionales durante la declaración. Entonces puedes omitirlos al llamar a la función.

/* radius puede ser omitido */
let drawCircle = (~color, ~radius=?, ()) => {
  setColor(color);
  switch (radius) {
  | None => startAt(1, 1)
  | Some(r_) => startAt(r_, r_)
  }
};

Cuando se da esta sintaxis, radius es ** englobado **con el tipo option de la biblioteca estándar, con el valor por defecto de None. Si es proporcionado, será englobado con Some. Entonces, el valor del tipo de radius es None o Some(int).

Nota: None | Some(foo) es un tipo de estructura de datos llamado variante, descrita anteriormente. Este particular tipo variante es proporcionado por la biblioteca estándar. Es llamado option. Su definición: type option('a) = None | Some('a).

Nota el unit () al final de drawCircle. Sin eso, como radius y color están ambos etiquetados, pueden ser curried, y pueden ser aplicados fuera de orden, no esta claro que significa lo siguiente:

let whatIsThis = drawCircle(~color);

¿Es whatIsThis una función drawCircle curried, esperando el parámetro opcional radius para ser aplicado? ¿O se terminó aplicando? Para hacer frente a esta confusión, añadir un argumento posicional (también conocido como no-etiquetado) a drawCircle (convencionalmente ()) y OCaml, como regla general, asumirá que el argumento opcional etiquetado es omitido cuando el argumento posicional es siempre proporcionado.

let curriedFunction = drawCircle(~color);
let actualResultWithoutProvidingRadius = drawCircle(~color, ());

Opcional pasado explicitamente

A veces, querrás enviar un valor a una función sin saber si el valor es None o Some(a). Ingenuamente, lo harías:

let result =
  switch (payloadRadius) {
  | None => drawCircle(~color, ())
  | Some(r) => drawCircle(~color, ~radius=r, ())
  };

Rápidamente esto se hace tedioso. Proporcionamos un atajo:

let result = drawCircle(~color, ~radius=?payloadRadius, ());

Esto significa "entiendo que radius es opcional, y que cuando le paso un valor necesita ser un int, pero no se si el valor que tengo que pasar es None o Some(val), así que te pasaré todo el option encapsulado".

Opcional con valor por defecto

A los argumentos etiquetados opcionales también se les puede proporcionar un valor por defecto. En este caso, no son encapsulados en un tipo option.

let drawCircle = (~radius=1, ~color, ()) => {
  setColor(color);
  startAt(radius, radius)
};

Funciones recursivas

Por defecto, un valor no puede ver un binding que le apunte, pero incluyendo la palabra clave rec en un binding let hace esto posible. Esto permite que las funciones se llamen y se vean a si mismas, dándonos el poder de la recursión.

let rec neverTerminate = () => neverTerminate();

Funciones mutuamente recursivas

Las funciones mutuamente recursivas comienzan como una única función recursiva usando la palabra clave rec, y entonces son todas encadenadas juntas con and:

let rec callSecond = () => callFirst()
and callFirst = () => callSecond();

Nota que no haya punto y coma al finalizar la primera linea y que no haya let en la segunda linea.

Consejos y Trucos

Hoja de referencia de las sintaxis de función:

Declaración

/* función anónima. Listada para completar */
(x) => (y) => 1;
/* sugar de lo anterior */
(x, y) => 1;
/* asignar a un nombre */
let add = (x, y) => 1;

/* etiquetado */
let add = (~first as x, ~second as y) => x + y;
/* con sugar rompecabezas */
let add = (~first, ~second) => first + second;

/* etiquetado con valor opcional */
let add = (~first as x=1, ~second as y=2) => x + y;
/* con rompecabezas */
let add = (~first=1, ~second=2) => first + second;

/* opcional */
let add = (~first as x=?, ~second as y=?) => switch (x) {...};
/* con rompecabezas */
let add = (~first=?, ~second=?) => switch (first) {...};

Con anotaciones de tipo

/* función anónima */
(x: int) => (y: int): int => 1;
/* sugar de lo anterior */
(x: int, y: int): int => 1;
/* asignar a un nombre */
let add = (x: int, y: int): int => 1;

/* etiquetado */
let add = (~first as x: int, ~second as y: int) : int => x + y;
/* con sugar de rompecabezas */
let add = (~first: int, ~second: int) : int => first + second;

/* etiquetado con valor por defecto */
let add = (~first as x: int=1, ~second as y: int=2) : int => x + y;
/* con sugar de rompecabezas */
let add = (~first: int=1, ~second: int=2) : int => first + second;

/* opcional */
let add = (~first as x: option(int)=?, ~second as y: option(int)=?) : int => switch (x) {...};
/* con sugar de rompecabezas */
/* observar que la llamada pasaría un `int`, no `option int` */
/* dentro de la función, `first` y `second` son `option int`. */
let add = (~first: option(int)=?, ~second: option(int)=?) : int => switch (first) {...};

Aplicación

/* aplicación anónima. Listado solo para completar */
add(x)(y);
/* sugar para lo anterior */
add(x, y);

/* etiquetado */
add(~first=1, ~second=2);
/* con sugar de rompecabezas */
add(~first, ~second);

/* aplicación con valor por defecto. Igual que la aplicación normal */
add(~first=1, ~second=2);

/* aplicación opcional explicita */
add(~first=?Some(1), ~second=?Some(2));
/* with rompecabezas */
add(~first?, ~second?);

Con anotaciones de tipo

/* aplicación anónima */
add(x: int)(y: int);

/* etiquetado */
add(~first=1: int, ~second=2: int);
/* con sugar de rompecabezas*/
add(~first: int, ~second: int);

/* aplicación valor por defecto. Igual que la aplicación normal */
add(~first=1: int, ~second=2: int);

/* aplicación opcional explicita */
add(~first=?Some(1): option(int), ~second=?Some(2): option(int));
/* con sugar de rompecabezas */
add(~first: option(int)?, ~second: option(int)?);

Firma de Tipo autónomo

/* primer tipo de arg, segundo tipo de arg, tipo de retorno */
type foo = int => int => int;
/* sugar para lo anterior */
type foo = (int, int) => int;

/* etiquetado */
type foo = (~first: int, ~second: int) => int;

/* etiquetado con valor por defecto */
type foo = (~first: int=?, ~second: int=?) => int;

En archivos de la interfaz

Para anotar una función desde el archivo de implementación (.re):

let add: int => int => int;
/* sugar para el anterior */
let add: (int, int) => int;

Las mismas reglas que en la sección anterior, excepto que se reemplaza type foo = bar con let add: bar.

No confunda esto con realmente exportar un tipo en el archivo de la interfaz.**. let add: bar anota un valor existente bar desde el archivo de implementación. type foo = bar exporta un tipo con la misma forma desde el archivo de implementación.
