If-Else

Los ifs de Reason son expresiones; se evalúa el contenido de su cuerpo:

let message = if (isMorning) {
  "Good morning!"
} else {
  "Hello!"
};

Note: an if-else expression without the final else branch implicitly gives (). So this:

if (showMenu) {
  displayMenu()
};

básicamente es lo mismo que:

if (showMenu) {
  displayMenu()
} else {
  ()
};

Que es usualmente lo que esperarías; displayMenu() debería regresar unit también.

Aquí hay una otra manera de verlo. Esto es claramente incorrecto :

let result = if (showMenu) {
  1 + 2
};

Dará un error de tipo, diciendo básicamente que la rama else implícita tiene el tipo unit mientras que la rama if tiene el tipo int. Intuitivamente, esto tiene sentido: ¿Qué valor de result sería, si showMenu era false?

También tenemos endulzamiento ternario (ternary sugar).

let message = isMorning ? "¡Buenos días!" : "¡Hola!";

Uso

if-else y las expresiones ternarias se utilizan menos en Reason que en otros lenguajes; Coincidencia de patrones Pattern-matching, elimina una categoría entera de código que antes requería condicionales. Da preferencia a if-else si sólo tienes, por ejemplo, 2 ramas.

Decisiones de Diseño

Ternarios en Reason son solo un endulzamiento (sugar) para variantes bool y un switch:

switch (isMorning) {
| true => "¡Buenos días!"
| false => "¡Hola!"
}

Si lo pasas a través de refmt, obtendrás:

isMorning ? "¡Buenos días!" : "¡Hola!";
