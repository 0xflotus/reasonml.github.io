Let Binding

Un "let binding", en otros lenguajes, pueden llamarse "asignación o declaración de una variable". let le da un nombre a un valor. Pueden ser usados por el código que viene después de los mismos.

let saludo = "hola!";
let puntaje = 10;
let nuevoPuntaje = 10 + puntaje;
...

Bloque de Alcance

Bindings pueden limitar su alcance mediante {}.

if (mostrarSaludo) {
  let mensaje = "Disfrutando de la documentacion hasta ahora?";
  print_endline(mensaje);
};
/* `mensaje` no es accesible aca! */

Bindings son Inmutables

"Inmutable" como en, "no cambia". Una vez que un binding hace referencia a un valor, no puede hacer referencia a nada mas (al menos que explicitamente contenga un valor mutable, explicado mas adelante). Sin embargo, podrías crear un nuevo binding con el mismo nombre que ensombrece (shadows) el binding previo; desde ese punto en adelante, el binding se referirá al nuevo valor asignado.

let mensaje = "hola";
print_endline(mensaje); /* Imprime "hola" */
let mensaje = "chau";
print_endline(mensaje); /* Imprime "chau" */

Consejos y Trucos

Ya que los bindings están limitados mediante {}, puedes crear un scope anónimo alrededor de ellos:

let mensaje = {
  let parte1 = "hola";
  let parte2 = "mundo";
  parte1 ^ " " ^ parte2
};
/* `parte1` y `parte2` ¡no son accesibles aquí! */

Esto previene el mal uso de los bindings después de estas lineas.

Decisiones de diseño

Reason funciona gracias a OCaml. Un let binding, en sintaxis de OCaml, funciona así:

let a = 1 in
let b = 2 in
a + b

Esto puede ser leído conceptualmente en este formato:

let a = 1 in
  let b = 2 in
    a + b

Que en Reason es lo siguiente:

let a = 1;
let b = 2;
a + b;

Lo cual te puede recordar a:

/* Reason syntax */
(a) =>
  (b) =>
    a + b;

Aunque no son estrictamente lo mismo, afortunadamente puedes ver que let es solo una expresión y ¡se asemeja a una función! En Reason, hemos transformado in en ; para tener familiaridad visual; pero no dejes que eso oculte la elegancia subyacente de la expresiones.
