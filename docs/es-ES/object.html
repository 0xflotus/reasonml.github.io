Objeto

La mayor parte del tiempo en Reason, usarás los registros para agrupar valores con nombres. En ciertas situaciones de nicho, sin embargo, podrías utilizar una característica similar llamada objeto; son un poco más flexibles y vienen con diferentes compensaciones.

Si vienes de JavaScript, muy probablemente no estas buscando objetos planos de Reason. Salta directamente a la sección de Consejos y Trucos.

Uso

Declaración de tipo

Un objeto no necesita una declaración de tipo, aunque puede tener una. Es como un registro, excepto con un .:

type tesla = {
  .
  color: string
};

El punto al principio indica que se trata de un tipo de objeto "cerrado", que significa que un objeto basado en este tipo debe tener exactamente esta forma.

type car('a) = {
  ..
  color: string
} as 'a;

Dos puntos, también llamado una elisión, indica que es un tipo de objeto "abierto" y por lo tanto también puede contener otros valores y métodos. Un objeto abierto también es polimorfo y por lo tanto requiere un parámetro.

Creación

type tesla = {.
  drive: int => int
};

let obj: tesla = {
  val hasEnvy = ref(false);
  pub drive = (speed) => {
    this#enableEnvy(true);
    speed
  };
  pri enableEnvy = (envy) => hasEnvy := envy
};

Este objeto es del tipo de objeto tesla y tiene un método público drive. También contiene un método privado enableEnvy que sólo es accesible desde dentro del objeto.

Como puedes ver, un objeto de Reason también puede acceder a this. El comportamiento de this de un objeto JavaScript puede ser peculiar; Reason this siempre señala al mismo objeto correctamente.

En el ejemplo siguiente se muestra un tipo de objeto abierto que utiliza un tipo como parámetro. El parámetro de tipo de objeto es requerido para implementar todos los métodos del tipo de objeto abierto.

type tesla('a) = {
  ..
  drive: int => int
} as 'a;

let obj tesla({.drive: int => int, doYouWant: unit => bool}) = {
  val hasEnvy = ref(false);
  pub drive = (speed) => {
    this#enableEnvy(true);
    speed
  };
  pub doYouWant = () => hasEnvy^;
  pri enableEnvy = (envy) => hasEnvy := envy
};

Usted puede utilizar el objeto anterior así:

obj#doYouWant();

Consejos y trucos

Si vienes desde JavaScript, usted probablemente no está buscando objetos básicos (vanilla) de Reason, sino de BuckleScript objeto especial. Éstos son diferentes en que:

  • acceden a los campos a través de ##.
  • utilizan #= para la mutación, si el campo es mutable (usando bs.set).
  • siempre vienen con Js.t (como un parámetro de tipo para el tipo Js.t).
  • compilan a objetos JS reales.

Como son utilizados tan a menudo, Reason da al valor del objeto BS [%bs.obj {foo: bar}] un especial endulzamiento sintáctico (syntactic sugar): {"foo": bar}. Se ve como un registro entrecomillado, básicamente. Igualmente para los tipos: {."foo": string}
