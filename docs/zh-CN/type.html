Type!

类型是Reason的亮点! 在这里, 你会看到为什么这么多人对类型感到兴奋。

本节简要介绍了类型语法, 以便你能够在阅读后续的章节而不感到困惑。 有关类型的更高级主题可以在 更多类型 一节中找到。

批注

下面的let写法不包含任何书面类型：

let score = 10;

Reason知道 score 是 int类型, 由值 10 来判断。这称为 ** 推断 **。

但类型也可以通过选择明确地写下来：

let score: int = 10;

还可以将任何表达式括在圆括号中并对其进行批注:

let myInt = 5;
let myInt: int = 5;
let myInt = (5: int) + (4: int);
let add = (x: int, y: int) : int => x + y;
let drawCircle = (~radius as r: int) : unit => ...;

注意：在最后一行， (~radius as r: int)是一个标签参数，更多详细信息可以点击查看 .

别名

可以用其他名称引用类型。它们等同于：

type scoreType = int;
let x: scoreType = 10;

设计决策

Reason由 OCaml 支持，其类型系统已接受数十年的工程考验。以下是一些亮点：

  • ** 类型可以推断 **。 类型系统为您推断类型, 即使您不手动将它们写下来。 这加快了原型阶段的速度。 此外，编辑器功能（如 vscode 的 codelens ）会在您编写代码时显示所有类型。

  • ** 类型覆盖率始终为 100% **。我们不需要 “类型覆盖”工具！每段原因代码都有一个类型。

  • ** 类型系统完全可信 **。 这意味着, 只要代码编译良好, 每种类型都保证它不会对自身撒谎。 In a conventional, best-effort type system, just because the type says it's e.g. "an integer that's never null", doesn't mean it's actually never null. In contrast, a pure Reason program has no null bugs.
