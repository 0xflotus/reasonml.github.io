Function

完整的函数语法备忘清单在最后

你能相信我们到现在还未谈到函数？

函数通过箭头符号声明并返回表达式。

let greet = (name) => "Hello " ++ name;

这声明一个函数并将其赋给了greet，你可以这样进行调用：

greet("world!"); /* "Hello world!" */

多参数函数通过逗号进行分割：

let add = (x, y, z) => x + y + z;
add(1, 2, 3); /* 6 */

对于更长的函数，你可以将其包裹在方括号内：

let greetMore = (name) => {
  let part1 = "Hello";
  part1 ++ " " ++ name
};

无参数

函数总是会带有一个参数；但有时，我们使用函数的副作用，并不会传任何参数给它。 在其他语言中，我们在理论上传递“无参数”。 在 Reason 中，每个函数都有一个参数；这里我们最终会传递一个值为 () 给它，被称作“unit”。

/* 接收 & 解构 unit 参数 */
let logSomething = () => {
  print_endline("hello");
  print_endline("world")
};

/* 通过类型为 unit 的值调用函数 */
logSomething();

()是在 unit中一个完全普通且唯一可能的值，Reason 出于方便为其提供了特殊语法。

标签参数

多参函数——尤其参数是相同类型的——调用时会给人带来困惑。

let addCoordinates = (x, y) => {
  /* use x and y here */
};
/* ... */
addCoordinates(5, 6); /* which is x, which is y? */

In OCaml/Reason, you can attach labels to an argument by prefixing the name with the ~ symbol:

let addCoordinates = (~x, ~y) => {
  /* use x and y here */
};
/* ... */
addCoordinates(~x=5, ~y=6);

因为可以自动柯里化（稍后会详细说明），我们可以以任意顺序提供参数：

addCoordinates(~y=6, ~x=5);

The ~x part in the declaration means the function accepts an argument labeled x and can refer to it in the function body by the same name. 你还可以为了方便在函数体内通过不同的名称引用参数：

let drawCircle = (~radius as r, ~color as c) => {
  setColor(c);
  startAt(r, r);
  /* ... */
};

drawCircle(~radius=10, ~color="red");

实际上，(~radius)仅仅是对 (~radius as radius) 的缩写 (称为双关语).

下面是关于参数输入的语法：

let drawCircle = (~radius as r: int, ~color as c: string)=> ...;

Curring

Reason 函数能够自动地被部分调用:

let add = (x, y) => x + y;
let addFive = add(5);
let eleven = addFive(6);
let twelve = addFive(7);

实际上，之前的add只不过是语法糖：

let add = (x) => (y) => x + y;

OCaml 对其进行优化以避免任何时候都能够不必要的函数分配(朴实地讲，这里是两个函数)! 这样，我们就得到

  • 良好的语法
  • 免费 Currying (实际上，每个函数有一个单独的参数！)
  • 没有性能损耗

可选的标签参数

标签函数参数能够在声明期间标记为可选。而后你可以在调用函数时忽略它们。

/* radius can be omitted */
let drawCircle = (~color, ~radius=?, ()) => {
  setColor(color);
  switch (radius) {
  | None => startAt(1, 1)
  | Some(r_) => startAt(r_, r_)
  }
};

当在给定的语法时，radius被包装在标准库的option类型中，默认值为None。 若提供，其将会被包装在Some中。 因此 radius 的类型值要么是 None，要么是 Some(int)。

Note：None | Some(foo) 是一种之前所描述的称为 variant 的数据结构类型， 特殊的 variant 类型由标准库提供。 其被称为option。 它的定义：type option('a) = None | Some('a)。

Note unit () 在 drawCircle 的末尾。 缺少它，由于 radius 和 color 都是标签参数，能够被柯里化，并能不按顺序调用，则对于下面代码的意义则不清晰：

let whatIsThis = drawCircle(~color);

whatIsThis 是一个等待可选的 radius 参数应用的柯里化的 drawCircle 函数？ 或已经完成应用？ 为了处理这一困惑，添加一个位置 (亦称 非标签) 参数到 drawCircle (一般为 ())，OCaml 也会，一般来说，当提供了可选参数时，会假定可选的标记参数会被忽略。

let curriedFunction = drawCircle(~color);
let actualResultWithoutProvidingRadius = drawCircle(~color, ());

显示传递可选参数

有时，你可能想要将某个值传递给函数却不知道这一个值是None 或 Some(a). 简单底，你可以这么处理：

let result =
 switch (payloadRadius) {
 | None => drawCircle(~color, ())
 | Some(r) => drawCircle(~color, ~radius=r, ())
 };

这很快就变得十分冗长。我们提供了一个快捷方式：

let result = drawCircle(~color, ~radius=?payloadRadius, ());

这意味着”我理解 radius 是可选的，当我传递给它一个值时需要是 int 类型，但我不确定我穿的值是 None 还是 Some(val)，因此我将会传递给你整个 option 包装器。

带默认值的可选参数

可选的标签参数也可以提供一个默认值。这一情况下，它们不是包装在 option 类型中。

let drawCircle = (~radius=1, ~color, ()) => {
  setColor(color);
  startAt(radius, radius)
};

递归函数

默认情况，值是无法看到指向自身的绑定，但若在 let 绑定中包含了 rec 关键字则使这成为可能。 这允许函数通过递归赋予的能力看到和调用它们自身。

let rec neverTerminate = () => neverTerminate();

互相递归函数

互相递归函数从一个使用 rec 的关键字单独的递归函数开始，然后通过 and 链接在一起：

let rec callSecond = () => callFirst()
and callFirst = () => callsecond();

注意第一行没有以分号结尾而第二行没有 let。

提示 & 技巧

函数语法备忘清单：

声明

/* anonymous function. Listed for completeness only */
(x) => (y) => 1;
/* sugar for the above */
(x, y) => 1;
/* assign to a name */
let add = (x, y) => 1;

/* labeled */
let add = (~first as x, ~second as y) => x + y;
/* with punning sugar */
let add = (~first, ~second) => first + second;

/* labeled with default value */
let add = (~first as x=1, ~second as y = 2) => x + y;
/* with punning */
let add = (~first=1, ~second=2) => first + second;

/* optional */
let add = (~first as x=?, ~second as y=?) => switch (x) {...};
/* with punning */
let add = (~first=?, ~second=?) => switch (first) {...};

带类型标记

/* anonymous function */
(x: int) => (y: int): int => 1;
/* sugar for the above */
(x: int, y: int): int => 1;
/* assign to a name */
let add = (x: int, y: int): int => 1;

/* labeled */
let add = (~first as x: int, ~second as y: int) : int => x + y;
/* with punning sugar */
let add = (~first: int, ~second: int) : int => first + second;

/* labeled with default value */
let add = (~first as x: int=1, ~second as y: int=2) : int => x + y;
/* with punning sugar */
let add = (~first: int=1, ~second: int=2) : int => first + second;

/* optional */
let add = (~first as x: option(int)=?, ~second as y: option(int)=?) : int => switch (x) {...};
/* with punning sugar */
/* note that the caller would pass an `int`, not `option int` */
/* Inside the function, `first` and `second` are `option int`. */
let add = (~first: option(int)=?, ~second: option(int)=?): int => switch (first) {...};

应用

/* anonymous application. Listed for completeness only */
add(x)(y);
/* sugar for the above */
add(x, y);

/* labeled */
add(~first=1, ~second=2);
/* with punning sugar */
add(~first, ~second);

/* application with default value. Same as normal application */
add(~first=1, ~second=2);

/* explicit optional application */
add(~first=?Some(1), ~second=?Some(2));
/* with punning */
add(~first?, ~second?);

带类型标记

/* anonymous application */
add(x: int)(y: int);

/* labeled */
add(~first=1: int, ~second=2: int);
/* with punning sugar */
add(~first: int, ~second: int);

/* application with default value. Same as normal application */
add(~first=1: int, ~second=2: int);

/* explicit optional application */
add(~first=?Some(1): option(int), ~second=?Some(2): option(int));
/* with punning sugar */
add(~first: option(int)?, ~second: option(int)?);

单独的类型签名

/* first arg type, second arg type, return type */
type foo = int => int => int;
/* sugar for the above */
type foo = (int, int) => int;

/* labeled */
type foo = (~first: int, ~second: int) => int;

/* labeled */
type foo = (~first: int=?, ~second: int=?, unit) => int;

接口文件

从实现文件(.re)中标记函数：

let add: int => int => int;
/* sugar for the above */
let add: (int, int) => int;

与上一节规则相同，除了将 type foo = bar 替换为 let add: bar。

不要对类型文件里实际导出的类型感到困惑。 let add: bar在具体实现的文件中标记了一个已有的值bar。 type foo = bar从具体实现文件中导出了一个相同模型的类型。
