Boolean

布尔类型用bool声明，取值为true或false。其常见操作有：

  • &&：逻辑与
  • ||：逻辑或
  • !：逻辑非
  • <=，>=，<，>
  • ==：值相等，深度比较数据结构：(1, 2) == (1, 2)返回true.。便捷但要小心使用
  • ===：引用相等，浅层对比。(1, 2) === (1, 2)返回false。let myTuple = (1, 2); myTuple === myTuple返回true。
  • !=：值不相等
  • !==：引用不相等

用法

注意：BuckleScript提供绑定JavaScript的 true和false，这不同于Reason/OCaml的true和false！ 在没有恰当转换（Js.to_bool和Js.Boolean.to_js_boolean）时不要无差别地使用它们。

提示&技巧

按需使用值相等. 这很便捷，但也可能会意外地比较了两个深层嵌套的数据结构，并造成巨大的性能损失。 对“相等”的判断也不总是明确。 比如，一个foo的数据是否与一个惰性foo相等？ 理想情况下，应当是可扩展的。 将来会更新使其成为可能同时保证可靠；如果有兴趣，请查看模块化隐式。

设计决策

本节假定了解了变体的知识。如果是第一次阅读本指南，请稍后再来！

布尔是（仅概念上来说）变形的一个特殊情况：type bool = True | False。 就设计来说，这一概念漂亮地消除了在类型系统中硬编码一个布尔类型的需要。 与变体一样，其缺点在于构造函数编译成了可读性较差但速度更快的表示形式。 这就是为什么BuckleScript在后续阶段中缺乏将React的真/假编译为JavaScript的真/假的信息。
