Логічний тип

Логічний тип називається bool і приймає значення true або false. Найбільш вживані операції:

  • &&: логічне І
  • ||: логічне Або
  • !: логічне заперечення.
  • <=, >=, <, >
  • ==: structural equal, compares data structures deeply: (1, 2) == (1, 2) is true. Convenient, but use with caution
  • ===: порівняння по посиланях, виконує не глибокі порівняння. (1, 2) === (1, 2) є не правдою. let myTuple = (1, 2); myTuple === myTuple є правдою.
  • !=: structural unequal
  • !==: нерівність по посиланню

Використання

Note: JS true and false aren't the same as the Reason/BuckleScript true and false! Don't use them interchangeably without proper conversion using BuckleScript's Js.to_bool and Js.Boolean.to_js_boolean.

Поради та хитрощі

Use structural equal tastefully. Це зручно, але ви можете випадково порівняти структури даних з глибокою вкладеністю, що погіршить швидкодію вашої програми. Також не завжди зрозуміло, що означає «однаковий». Наприклад, чи є значення foo ідентичним до лінивого foo? В ідеалі було б гарно, якщо б це можна було підключати. В майбутньому ми плануємо зробити це можливим і надійним; якщо вам цікаво, то прочитайте modular implicit.

Проектні рішення

В цій секції вважається, що ви знаєте варіанти. Якщо ви вперше читаєте цю документацію, то можете повернуться до неї пізніше!

Логічний тип (але тільки концептуально) - це спеціальний випадок варіанту type bool = True | False. В контексті дизайну, це елегантно усуває потребу в визначенні логічного типу в системі типів. Недоліком є те, що як і інші варіанти, конструктори компілюються в менш читабельний, але швидкий код. Саме тому BuckleScript не достає інформації, на пізніших етапах, щоб скомпілювати true/false із Reason в true/false з JavaScript.
