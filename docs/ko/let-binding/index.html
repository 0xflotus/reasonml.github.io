Let Binding

A "let binding", in other languages, might be called a "variable declaration". let gives names to values. They can be seen and referenced by code that comes after them.

let greeting = "hello!";
let score = 10;
let newScore = 10 + score;

블럭 스코프

바인딩은 {}를 통해 스코프가 설정됩니다.

if (displayGreeting) {
  let message = "Enjoying the docs so far?";
  print_endline(message)
};
/* `message` 는 더 이상 접근할 수 없습니다! */

바인딩은 불변입니다

"불변"은 "변하지 않음"을 나타냅니다. 하나의 바인딩이 한 값을 나타내면, 이 바인딩은 다른 어떠한 값도 나타낼 수 없습니다 (명시적으로 변하는 값을 담는다고 하지 않을 경우에, 추후에 논의). 하지만, 당신은 이전의 바인딩을 가리는 방식으로 같은 이름으로 바인딩을 만들 수 있습니다. 해당 코드 이후부터 그 바인딩은 새로 할당된 값을 가리킬 것입니다.

let message = "hello";
print_endline(message); /* "hello"를 출력 */
let message = "bye";
print_endline(message); /* "bye"를 출력 */

팁 & 트릭

바인딩은 {}로 스코프가 설정되므로 익명 스코프를 만들고 감싸서 다음과 같이 할 수 있습니다.

let message = {
  let part1 = "hello";
  let part2 = "world";
  part1 ++ " " ++ part2
};
/* 여기서는 `part1` 과 `part2` 에 접근할 수 없습니다! */

이는 해당 라인보다 뒤에서 바인딩이 잘못 사용되는 것을 사전에 막을 수 있게 합니다.

디자인 결정

Reason is backed by OCaml under the hood. A let binding, in OCaml syntax, looks like this:

let a = 1 in
let b = 2 in
a + b

우리는 개념 상, 이런 식으로 읽을 수 있습니다:

let a = 1 in
  let b = 2 in
    a + b

Reason에서는 다음과 같습니다:

let a = 1;
let b = 2;
a + b;

이는 다음을 상기시켜줄 수 있습니다:

/* Reason 문법 */
(a) =>
  (b) =>
    a + b;

Though they're not strictly the same, hopefully you can see that let is just an expression and akin to a function! In Reason, we've turned in into ; for visual familiarity; but don't let that hide the underlying elegance of expressions.
