If-Else

Reason의 if는 식입니다, if는 몸통의 요소에 의해 평가됩니다:

let message = if (isMorning) {
  "Good morning!"
} else {
  "Hello!"
};

주의 if-elseif-else 식이 마지막에 else 브랜치로 끝나지 않을 경우 암시적으로 ()를 반환합니다.

if (showMenu) {
  displayMenu()
};

는 기본적으로 다음과 같습니다.

if (showMenu) {
  displayMenu()
} else {
  ()
};

예상하신대로 displayMenu()는 unit을 반환합니다.

Here's another way too look at it. This is clearly wrong:

let result = if (showMenu) {
  1 + 2
};

이 식은 타입 오류를 냅니다. 암시적으로 존재하던 else 브랜치는 unit을 반환하므로 if 브랜치가 반환하는 int와 타입이 다르기 때문입니다. Intuitively, this makes sense: what would result's value be, if showMenu was false?

또 Reason에는 삼항식이라는 문법적 설탕이 있습니다.

let message = isMorning ? "Good morning!" : "Hello!";

사용

if-else and ternary are much less used in Reason than in other languages; Pattern-matching kills a whole category of code that previously required conditionals. Prefer if-else if you only have, say, 2 branches.

디자인 결정

Reason ternary is just a sugar for the bool variant and a switch:

switch (isMorning) {
| true => "Good morning!"
| false => "Hello!"
}

If you pass that through refmt, you'd get:

isMorning ? "Good morning!" : "Hello!";
