타입!

타입은 Reason의 강점입니다! 여기서, 당신은 왜 그렇게 많은 사람들이 이것에 열광하는지 경험할 수 있을 것입니다.

이 섹션에선 타입 문법을 간단히 소개함으로써 이후의 섹션을 헷갈림 없이 나아갈 수 있도록 합니다. 타입에 대한 더 심층적인 주제는 More On Type 섹션에서 볼 수 있습니다.

명시

이 let 바인딩은 명시적 타입을 포함하고 있지 않습니다.

let score = 10;

Reason은 score가 10이라는 값을 바탕으로 타입이 int라는 것을 압니다. 이것은 추론이라고 불립니다.

하지만 타입은 명시적으로도 작성될 수 있습니다:

let score: int = 10;

또 당신은 표현식을 괄호 속에 감싸고 명시를 할 수도 있습니다:

let myInt = 5;
let myInt: int = 5;
let myInt = (5: int) + (4: int);
let add = (x: int, y: int) : int => x + y;
let drawCircle = (~radius as r: int) : unit => ...;

참고: 마지막 줄에서, (~radius as r: int)는 이름 있는 인자입니다. 이에 대해 더 구체적인 설명은 이 곳에 있습니다.

별칭

당신은 어떤 타입을 다른 이름으로 참조할 수 있습니다. 그들은 동일합니다:

type scoreType = int;
let x: scoreType = 10;

디자인 결정

Reason은 타입 시스템이 수십년간 발전해온 OCaml을 기반으로 합니다. 다음과 같은 강점이 존재합니다:

  • 타입은 추론 가능합니다. 타입 시스템은 당신이 직접 타입을 명시하지 않더라도 타입을 추론합니다. 이것은 프로토타입을 빠르게 할 수 있도록 합니다. 추가로, VSCode's codelens와 같은 에디터 기능은 당신이 코드를 작성하는 중에 모든 타입을 보여줍니다.

  • 타입 범위는 항상 100% 입니다.. "타입 범위" 도구는 필요하지 않습니다. Reason의 모든 코드는 타입을 가집니다.

  • 타입 시스템은 완전히 "안전"합니다. 이것은, 당신의 코드가 컴파일 되는 이상, 모든 타입은 자신에 대해 거짓말하지 않는다는 것을 보장합니다. 종래의 타입 시스템에서는, 그저 타입이 "절대 null이 아닌 정수" 라고 한다고 해서 그것이 실제로 절대 null이 아닌 것은 아닙니다. 반면에 순수한 Reason 프로그램은 null 버그가 없습니다.
