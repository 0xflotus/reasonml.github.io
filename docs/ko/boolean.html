부울

부울은 bool이라는 타입을 가지며 true 혹은 false가 될 수 있습니다. 기본 연산은 다음이 있습니다:

  • &&: 논리곱
  • ||: 논리합
  • !: 부정
  • <=, >=, <, >
  • ==: structural equal, compares data structures deeply: (1, 2) == (1, 2) is true. Convenient, but use with caution
  • ===: 레퍼런스 동치, 얕게 비교합니다. (1, 2) === (1, 2) 은 false입니다. let myTuple = (1, 2); myTuple === myTuple 은 true입니다.
  • !=: structural unequal
  • !==: 레퍼런스 동치 부정

사용

Note: JS true and false aren't the same as the Reason/BuckleScript true and false! Don't use them interchangeably without proper conversion using BuckleScript's Js.to_bool and Js.Boolean.to_js_boolean.

팁 & 트릭

Use structural equal tastefully. 이는 편리하지만, 어떤 경우엔 뜻하지 않게 두 개의 깊은 데이터 구조를 비교하게 하여 큰 성능 하락을 초래하게 될 수 있습니다. 또한 어떤 것이 "동치"로 여겨지는지도 항상 자명하지 않습니다. 예를 들어, 하나의 데이터 foo는 평가가 지연된(lazy) foo와 같을까요? 이상적으로는 이것에 대한 확장이 가능해야 합니다. 이것이 가능하고 믿을 수 있게 하는 변화가 만들어지고 있습니다. 관심이 있으시다면, modular implicit를 확인해보세요.

디자인 결정

이 섹션은 variants에 대한 지식을 전제로 합니다. 만약 이 가이드를 처음 읽는 것이라면, 나중에 여기로 돌아오셔도 괜찮습니다.

Boolean은 (오직 개념적으로만) variant의 특수한 경우입니다: type bool = True| False. 디자인의 관점에서, 이것은 타입 시스템에서 부울 타입을 하드 코딩할 필요성을 없애줍니다. 문제점은, 다른 variant들과 마찬가지로, 생성자들이 가독성은 떨어지지만 더 빠른 표현으로 컴파일 된다는 것입니다. 이것은 BuckleScript가 나중 단계에서 Reason의 true/false를 자바스크립트의 true/false로 변환하기 위해 필요한 정보가 부족한 이유입니다.
