함수

문서의 끝에 전체 함수 문법 치트 시트가 있습니다

아직도 함수를 다루지 않았다는 것이 믿겨지시나요?

함수는 화살표와 함께 선언되며 식을 반환합니다.

let greet = (name) => "Hello " ++ name;

이 선언은 greet이라는 이름에 함수를 할당하고, 이는 다음과 같이 호출할 수 있습니다:

greet("world!"); /* "Hello world!" */

다인자 함수는 쉼표로 구분된 인자를 가집니다:

let add = (x, y, z) => x + y + z;
add(1, 2, 3); /* 6 */

긴 함수를 선언하려면 함수 몸통을 블럭으로 감싸면 됩니다:

let greetMore = (name) => {
  let part1 = "Hello";
  part1 ++ " " ++ name
};

무인자 함수

함수는 언제나 인자를 가집니다. 하지만 때때로 부수효과를 가지면서 인자로 넘길 것이 딱히 없는 함수도 있습니다. 다른 언어였다면 우리는 개념적으로 "텅 빈 인자"를 넘겼을 것입니다. 하지만 Reason의 모든 함수는 인자를 가져야합니다. 따라서 우리는 관습적으로 "unit"이라 불리는 ()라는 값을 넘깁니다.

/* unit 인자를 받고 destructure */
let logSomething = () => {
  print_endline("hello");
  print_endline("world")
};

/* unit 타입을 가진 값으로 함수 호출 */
logSomething();

()는 일반적인 값입니다. 또한, unit이 될 수 있는 유일한 값이기도 합니다. Reason은 편리함을 위해 특수한 문법을 부여했습니다.

이름 있는 인자

다인자 함수, 특히 같은 타입의 인자들을 가진 다인자 함수는 호출시에 헷갈릴 수 있습니다.

let addCoordinates = (x, y) => {
  /* use x and y here */
};
/* ... */
addCoordinates(5, 6); /* which is x, which is y? */

OCaml/Reason에선, 인자에 이름을 붙일 수 있습니다:

let addCoordinates = (~x, ~y) => {
  /* use x and y here */
};
/* ... */
addCoordinates(~x=5, ~y=6);

Reason은 커링을 지원하므로(밑에 구체적인 설명이 있습니다), 인수를 어떤 순서로든 배열해도 상관 없습니다:

addCoordinates(~y=6, ~x=5);

The ~x part in the declaration means the function accepts an argument labeled x and can refer to it in the function body by the same name. 간결함을 위해, 함수 몸통에서 인자를 또 다른 이름으로 사용할 수 있습니다:

let drawCircle = (~radius as r, ~color as c) => {
  setColor(c);
  startAt(r, r);
  /* ... */
};

drawCircle(~radius=10, ~color="red");

사실은, (~radius) 는 그저 (~radius as radius)의 줄임(punning이라 불림) 일 뿐입니다.

인자 타입의 명시는 다음과 같이 할 수 있습니다:

let drawCircle = (~radius as r: int, ~color as c: string) => ...;

커링

Reason의 함수는 부분적으로 호출될 수 있습니다:

let add = (x, y) => x + y;
let addFive = add(5);
let eleven = addFive(6);
let twelve = addFive(7);

실제로, 위의 add 함수는 다음 코드의 문법적 설탕에 불과합니다:

let add = (x) => (y) => x + y;

OCaml은 가능한 언제든 불필요한 함수 할당을 피하기 위한 (정확히는 2개의 함수가 존재합니다) 최적화를 수행합니다. 이것으로 다음과 같은 이점을 얻습니다:

  • 좋은 문법
  • 성능 손실 없는 커링 (사실 모든 함수는 하나의 인자만을 가집니다!)
  • 성능 오버헤드 없음

선택적 이름 있는 인자

선언에서 이름 있는 인자를 선택적이게 만들 수 있습니다. 함수를 호출할 때 이들을 생략할 수 있습니다.

/* radius 는 생략될 수 있음 */
let drawCircle = (~color, ~radius=?, ()) => {
  setColor(color);
  switch (radius) {
  | None => startAt(1, 1)
  | Some(r_) => startAt(r_, r_)
  }
};

이 문법에서, radius는 표준 라이브러리의 option 타입에 싸여집니다. 값은 기본적으로 None이 됩니다. 만약 인수가 주어졌다면, 값은 Some에 싸여졌을 것입니다. 따라서 radius의 타입은 None 또는 Some(int)입니다.

참고: None | Some(foo)는 variant라고 불리는 데이터 구조이며 여기 설명되어 있습니다. 이 variant 타입은 표준 라이브러리가 제공합니다. 이는 option이라 불립니다. 이것의 정의는 type option('a) = None | Some('a) 입니다.

drawCircle의 끝에 있는 unit ()에 주목하세요. 이것이 없으면, radius와 color 둘 다 이름이 붙여졌고, 커링될 수 있으며, 순서에 상관없이 적용될 수 있으므로 다음이 무엇인지 모호하게 됩니다:

let whatIsThis = drawCircle(~color);

whatIsThis는 선택적 인자인 radius가 적용되기를 기다리는 커리된 drawCircle 함수 인가요? 또는 적용이 끝난 함수인가요? 이 혼란을 바로잡기 위해선, drawCircle 끝에 고정된(즉 이름이 붙여지지 않은) 인자(관습적으로 ()) 를 붙이면 경험에 기반한 규칙에 의해 OCaml이 고정된 인자가 주어지면 선택적 인자가 생략된 것으로 간주합니다.

let curriedFunction = drawCircle(~color);
let actualResultWithoutProvidingRadius = drawCircle(~color, ());

명시적으로 전달된 선택적 인자

가끔, 값이 None인지 Some(a)인지에 상관없이 함수에 넘겨야 할 때가 있습니다. 단순하게는 다음과 같이 할 것입니다:

let result =
  switch (payloadRadius) {
  | None => drawCircle(~color, ())
  | Some(r) => drawCircle(~color, ~radius=r, ())
  };

이건 꽤 싫증납니다. Reason은 손쉬운 대안을 제공합니다:

let result = drawCircle(~color, ~radius=?payloadRadius, ());

이것은 "난 radius가 선택적이라는 것을 알고, 값을 넘길때 int가 돼야 하지만, 내가 넘기는 값이 None인지 Some(val)인지 모르니까 option 래퍼를 통째로 넘길게." 라는 의미입니다.

기본 값이 있는 선택적 인자

선택적 이름 붙여진 인자에게 기본값을 부여할 수 있습니다. 이 경우, 인자는 option 타입에 싸여있지 않습니다.

let drawCircle = (~radius=1, ~color, ()) => {
  setColor(color);
  startAt(radius, radius)
};

재귀 함수

기본적으로, 값은 자기를 가르키는 바인딩을 볼 수 없습니다. 하지만 let 바인딩에 rec 키워드를 붙이면 이것이 가능해집니다. 이것은 함수가 자신을 보고 호출할 수 있게 하는 재귀의 힘을 줍니다.

let rec neverTerminate = () => neverTerminate();

서로 재귀적인 함수

서로 재귀적인 함수들은 rec 키워드를 사용하여 일반적인 재귀 함수처럼 시작하며, and 키워드를 사용하여 함께 묶여집니다.

let rec callSecond = () => callFirst()
and callFirst = () => callSecond();

첫 번째 줄의 끝에 세미콜론이 없으며 두 번째 줄에 let이 없다는 점을 주목하세요.

팁 & 트릭

함수 문법을 위한 치트 시트:

선언

/* 익명 함수. 완전함을 위해서만 리스트에 포함 */
(x) => (y) => 1;
/* 위에 대한 문법적 설탕 */
(x, y) => 1;
/* 이름 할당*/
let add = (x, y) => 1;

/* 이름 있는 인자 */
let add = (~first as x, ~second as y) => x + y;
/* punning 문법적 설탕 */
let add = (~first, ~second) => first + second;

/* 기본값과 함께 이름 붙여짐 */
let add = (~first as x=1, ~second as y=2) => x + y;
/* punning과 함께 */
let add = (~first=1, ~second=2) => first + second;

/* 선택적 인자 */
let add = (~first as x=?, ~second as y=?) => switch (x) {...};
/* punning과 함께 */
let add = (~first=?, ~second=?) => switch (first) {...};

타입 명시와 함께

/* 익명 함수 */
(x: int) => (y: int): int => 1;
/* 위에 대한 문법적 설탕 */
(x: int, y: int): int => 1;
/* 이름 할당 */
let add = (x: int, y: int): int => 1;

/* 이름 있는 인자 */
let add = (~first as x: int, ~second as y: int) : int => x + y;
/* punning 문법적 설탕과 함께 */
let add = (~first: int, ~second: int) : int => first + second;

/* 기본값과 함께 이름 붙여짐 */
let add = (~first as x: int=1, ~second as y: int=2) : int => x + y;
/* punning 문법적 설탕과 함께 */
let add = (~first: int=1, ~second: int=2) : int => first + second;

/* 선택적 인자 */
let add = (~first as x: option(int)=?, ~second as y: option(int)=?) : int => switch (x) {...};
/* punning 문법적 설탕과 함께 */
/* 호출자는 `option int`가 아닌 `int`를 넘길 것입니다 */
/* 함수 안에선 `first`와 `second`는 `option int` 입니다. */
let add = (~first: option(int)=?, ~second: option(int)=?) : int => switch (first) {...};

적용

/* 익명 적용 완전함을 위해서만 리스트에 포함 */
add(x)(y);
/* 위에 대한 문법적 설탕 */
add(x, y);

/* 이름 있는 인자 */
add(~first=1, ~second=2);
/* punning 문법적 설탕과 함께 */
add(~first, ~second);

/* 기본값이 있는 적용. 일반 적용과 같음 */
add(~first=1, ~second=2);

/* 명시적 선택적 인자 적용 */
add(~first=?Some(1), ~second=?Some(2));
/* punning과 함께 */
add(~first?, ~second?);

타입 명시와 함께

/* 익명 적용 */
add(x: int)(y: int);

/* 이름 있는 인자 */
add(~first=1: int, ~second=2: int);
/* punning 문법적 설탕과 함께 */
add(~first: int, ~second: int);

/* 기본값이 있는 적용. 일반 적용과 같음 */
add(~first=1: int, ~second=2: int);

/* 명시적 선택적 인자 적용 */
add(~first=?Some(1): option(int), ~second=?Some(2): option(int));
/* punning 문법적 설탕과 함께 */
add(~first: option(int)?, ~second: option(int)?);

독립된 타입 시그니쳐

/* 첫번째 인자 타입, 두번째 인자 타입, 반환값 타입 */
type foo = int => int => int;
/* 위에 대한 문법적 설탕 */
type foo = (int, int) => int;

/* 이름 있는 인자 */
type foo = (~first: int, ~second: int) => int;

/* 기본값과 함께 이름 붙여짐 */
type foo = (~first: int=?, ~second: int=?) => int;

인터페이스 파일에서

구현 파일(.re)에 있는 함수를 명시하려면:

let add: int => int => int;
/* 위에 대한 문법적 설탕 */
let add: (int, int) => int;

type foo = bar를 let add: bar로 바꿔야 한다는 것만 제외하면 같은 규칙이 적용됩니다.

이것을 인터페이스 타입에서 실제로 타입을 export 하는 것과 헷갈리지 마세요. let add: bar는 구현 파일에 이미 존재하는 bar 값을 명시합니다. type foo = bar는 구현 파일에서 같은 형태를 가진 타입을 export 합니다.
