Browsererweiterung: Reason-tools

Reason-tools ermöglicht es dir, schnell zwischen der OCaml-Syntax und der Reason-Syntax zu wechseln, wenn du Tutorials und Dokumentationen durchstöberst, die in einer dieser Syntaxen geschrieben wurden.

BuckleScript

Dir wird BuckleScript häufig im restlichen Teil dieser Dokumentation begegnen, da es die Engine ist, die unsere JS-Kompilation antreibt. Dessen globale Binärdateien werden durch npm install -g bs-platform installiert.

Andere Hilfsmittel

Die globale Installation, die du im vorigen Abschnitt durchgeführt hast, bringt ein paar zusätzliche nützliche Werkzeuge mit sich, die wir hier beschreiben.

Refmt

refmt ("Reason format") ist das Programm, das das Autoformatierungs-Feature unserer Editoren antreibt. Es nimmt deinen Code auf und spuckt ihn schön formatiert aus. Es dient auch dazu, zu/von der Reason/OCaml-Syntax zu konvertieren. Die Reason-Community verwendet es, um in verschiedenen Projekten einen konsistenten Codestil zu erreichen und zeitaufwändiges manuelles Formatieren & und Debatten über Stile zu vermeiden.

Es formatiert den Code responsiv abhängig von der Breite des Editors. Mit anderen Worten bricht es nicht nur naiv zur nächsten Zeile bei einer bestimmten Zeichengrenze um; Es löst die Layout-Einschränkungen und ordnet deinen Code entsprechend an.

refmt kann direkt im Terminal eingesetzt werden. Zum Beispiel, um deinen Code außerhalb des Editors zu formatieren, verwende refmt --in-place myFile.re. Siehe refmt --help für alle Optionen.

Merlin

Merlin ist die zugrundeliegende Engine, die Typhinweise, Refactorings, Echtzeitfehlermeldungen, Sprung zu Definitionen, usw. für unsere Editoren antreibt. Ihr Kommandozeilennamme ist ocamlmerlin, obwohl du sie nicht manuell aufrufen würdest (Editoren starten sie selbstständig und interagieren mit ihr).

Um Merlin so zu konfigurieren, dass es dein Projekt versteht, musst du eine .merlin-Datei im Wurzelverzeichnis des Projekts erstellen (Dokumentation hier). Für den JS-Workflow wird diese Konfiguration automatisch durch BuckleScripts bsb erzeugt.

REPL

Reason kommt mit einer REPL namens rtop, die dich Code auf interaktive Weise ausführen lässt. Sie beinhaltet intelligente, typgetriebene Autovervollständigung.

Verwende #quit;, um deine REPL-Sitzung zu beenden.

Beachte, dass rtop momentan nicht leicht mit Paketen und external zusammenarbeitet. Wir empfehlen, den Code innerhalb unseres Try-Spielplatzes auszuwerten.

re:bench

re:bench ist ein Spielplatz für das Online-Benchmarking. Es unterstützt Testfälle, die in Reason, OCaml und JavaScript geschrieben wurden, und ermöglicht das Teilen von Benchmarks über eindeutige URLs. Es wird nach JS durch BuckleScript kompiliert.

Redex

Redex: das Paketverzeichnis für Reason.

ocamlc, ocamlopt, ocamlrun

ocamlc und ocamlopt sind die nackten OCaml-Compiler.
