Mutation

Reason hat großartige Fähigkeiten für traditionelle imperative & mutative Programmierung. Du solltest diese Features spärlich einsetzen aber manchmal ermöglichen sie, dass dein Code performanter wird und ihn in einem vertrauten Muster zu schreiben.

Let-Bindung mutieren

Let-Bindungen sind unveränderlich, aber du kannst sie mit ref umgeben, was wie eine Box ist, deren Inhalt sich ändern kann:

let foo = ref(5);

Verwendung

Du kannst den eigentlichen Wert einer ref durch den Postfix-Operator ^ erhalten:

let five = foo^; /* 5 */

Weise foo so einen neuen Wert zu:

foo := 6;

Beachte dass die vorige Bindung five mit 5 gleich bleibt, denn sie enthält das zugrundeliegende Elemente in der ref-Box und nicht die ref selbst.

Tip & Tricks

Spaß beiseite! ref ist eigentlich kein spezielles Feature! Es ist bloß gewöhnlicher syntaktischer Zucker für einen vordefinierten, mutierbaren Record-Typen namens ref in der Standardbibliothek (suche dort auf der Seite nach „References“). Hier ist eine Version ohne syntaktischen Zucker:

let foo = {contents: 5};
let five = foo.contents;
foo.contents = 6;

Bevor du zu ref greifst, nehme zur Kenntnis, dass du leichtgewichtige, lokale „Mutationen“ durch Überschreiben von let-Bindungen erreichen kannst:

let foo = 10;
let foo = someCondition ? foo + 5 : foo;
print_int(foo); /* either 15 or 10 */
