String & Char

String

In Reason werden Strings durch doppelte Anführungszeichen umgeben (einfache Anführungszeichen sind für den Zeichentyp unten reserviert).

let greeting = "Hello world!";
let multilineGreeting = "Hello
 world!";

Sonderzeichen im String müssen maskiert werden:

let oneSlash = "\\";

Um Strings zusammenzusetzen, verwende ++:

let greetings = "Hello " ++ "world!";

Quoted String

Es gibt eine spezielle Syntax für Strings, die

  • mehrzeilige Strings wie zuvor erlaubt,
  • keine Maskierung für Sonderzeichen erfordert,
  • Ansatzpunkte für spezielle Präprozessoren bietet.
let greetingAndOneSlash = {|Hello
World
\
Hehe...
|};

Analog gesprochen ist es wie die String-Interpolation von JavaScript, außer dass Sonderzeichen nicht maskiert werden müssen und ohne enthaltene Interpolation von Variablen. Dennoch ist es trivial, letztere Funktionalität wiederherzustellen, wie in BuckleScript:

let world = {js|世界|js}; /* Supports Unicode characters */
let helloWorld = {j|你好，$world|j}; /* Supports Unicode and interpolation variables */

Der spezielle Präprozessor von BuckleScript kann dann nach Strings suchen, die von solchen Markierungen mit js und j umgeben sind, und diese in etwas anderes umformen.

Verwendung

Weitere String-Operationen können in der Standardbibliothek gefunden werden. Für die JS-Kompilierung, siehe die bekannte js.String-API in den BuckleScript API docs. Da ein Reason-String auf einen JavaScript-String abgebildet wird, kannst du die String-Operationen in beiden Standardbibliotheken miteinander kombinieren.

Tips & Tricks

https://twitter.com/jusrin00/status/875238742621028355

Du hast nun ein ausdrucksstarkes Typsystem! In einer ungetypten Sprache würdest du die Bedeutung eines Strings oft überladen, indem du ihn verwendest als:

  • eine eindeutige ID: var BLUE_COLOR = "blue"
  • einen identifizierenden Bezeichner in einer Datenstruktur: var BLUE = "blue"; var RED = "red"; var colors = [BLUE, RED]
  • den Namen einer Objekteigenschaft: person["age"] = 24
  • ein Enum: if (audio.canPlayType() === 'probably') {...} (ಠ_ಠ)
  • andere verrückte Muster, die du bald schrecklich finden wirst, nachdem du dich an Reasons Alternativen gewöhnt hast.

Je mehr du den schlechten String-Typen überlädst, desto weniger kann das Typsystem (oder ein Teamkollege) dir helfen! Reason bietet prägnante, schnelle und wartbare Alternativen bei Typen & Datenstrukturen zu den oben genannten Anwendungsfällen (z. B. Varianten, in einem späteren Abschnitt).

Bei der nativen Kompilierung werden Reason-Strings zu einer einfachen Darstellung kompiliert, deren Performance einfach zu analysieren ist, wobei manchmal eine manuelle Leistungsoptimierung erforderlich ist. Zum Beispiel, Strings wie "hi " ++ "how " ++ "are " ++ "you?" auf naive Weise zusammenzusetzen erzeugt zwischenzeitlich die Strings "are you?" und "how are you?" (obwohl diese einfachen Fälle zu einer einzigen Zeichenfolge optimiert werden könnten). In diesem Fall ist String.concat vorzuziehen. In gewisser Weise ist es ganz nett, dass die traditionelle Laufzeitanalyse, die wir in der Schule gelernt haben, endlich wieder nützlich sein kann.

Bei der JavaScript-Kompilierung wird ein Reason-String einem JavaScript-String und umgekehrt zugeordnet, sodass keine der oben genannten Belange oder Analysemöglichkeiten bestehen.

Designentscheidungen

Das Feature bei Quoted Strings, Sonderzeichen nicht maskieren zu müssen, ermöglicht gepflegte DSLs wie reguläre Ausdrücke:

let r = Str.regexp({|hello \([A-Za-z]+\)|});

im Gegensatz zu

let r = Str.regexp("hello \\([A-Za-z]+\\)");

Bei der JS-Kompilierung würdest du jedoch stattdessen [%bs.re] und Js.Re einsetzen, da Str nicht verfügbar ist.

Reason/OCamls Betonung von Einfachheit gegenüber Cleverness kann hier durch seine unkomplizierte native String-Implementierung gesehen werden. Eine übermäßig ausgeklügelte String-Implementierung kann manchmal nach hinten losgehen.

Char

Reason hat einen Typ für Strings, die aus einem einzigen Zeichen bestehen:

let firstLetterOfAlphabet = 'a';

Bemerkung: Char unterstützt kein Unicode oder UTF-8.

Tips & Tricks

Ein Zeichen kompiliert zu einer Ganzzahl zwischen 0 und 255, für extra Geschwindigkeit. Du kannst auf ihm auch Pattern-Matching (wird später behandelt) anwenden:

let isVowel = (theChar) =>
  switch (theChar) {
  | 'a'| 'e'| 'i'| 'o'| 'u'| 'y' => true
  | _ => false
  };

Um einen String in ein Char umzuwandeln, verwende "a".[0]. Um ein Char in einen String zu konvertieren, verwende String.make(1, 'a').
