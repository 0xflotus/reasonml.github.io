Tipo!

Tipos são o destaque de Reason! Aqui você verá o porquê deles serem tão excitantes.

Essa seção apresenta brevemente a sintaxe dos tipos para que você possa entender as próximas seções sem ficar confuso. Tópicos avançandos em tipos podem ser encontrados na seção Mais em Tipos.

Anotações

Um let-biding não apresenta nenhum tipo escrito:

let score = 10;

Reason sabe que score é um int, a partir do valor 10. Isto é chamado de inferência.

Porém tipos podem também ser explicitamente escritos por escolha:

let score: int = 10;

Você pode encapsular qualquer expressão com parenteses e anotá-la:

let meuInt = 5;
let meuInt: int = 5;
let meuInt = (5: int) + (4: int);
let soma = (x: int, y: int) : int => x + y;
let desenharCirculo = (~radius as r: int) : unit => ...;

Nota: na última linha, (~radius as r: int) é um argumento nomeado. Mais sobre isso aqui.

Aliases

Você pode se referir a um tipo usando um nome diferente. Isso seria equivalente:

type scoreType = int;
let x: scoreType = 10;

Decisões de Design

O backend de Reason é em OCaml, linguagem da qual o sistema de tipos teve décadas de engenharia. Aqui estão algumas notas:

  • Tipos podem ser inferidos. O sistema de tipos deduz o tipo para você mesmo se você não escrevê-lo manualmente. Isso acelera a fase de prototipagem. Adicionalmente, features de editores tais como o codelens do VSCode exibe todos os tipos enquanto você escreve o código.

  • A cobertura de tipos sempre é 100%. Nós não precisamos de uma ferramenta de "type coverage"! Cada parte de um código em Reason tem um tipo.

  • O sistema de tipos é completamente seguro (aka "sound") Isso significa que, desde que seu código compile bem, todo tipo é garantido não estar mentindo sobre si mesmo. Em um sistema de tipos convencional, até esforçadinho, não é porque o sistema de tipos que, por exemplo, o valor é um "inteiro nunca nulo" que ele realmente nunca é nulo. Em contrantes, um programa Reason puro não possui bug com valores nulos.
