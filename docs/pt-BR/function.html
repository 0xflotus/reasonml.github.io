Função

Cheat sheet com exemplos completo de syntax para criação de funções no final da página

Você acredita que ainda não vimos funções até agora?

Funções são declaradas com o uso de setas ( => ) e uma expressão de retorno.

let greet = (name) => "Hello " ++ name;

Este trecho declara uma função com o nome greet, podemos chama-la da seguinte forma:

greet("world!"); /* "Hello world!" */

Funcões com multiplos argumentos tem seus argumentos separados por vírgula:

let add = (x, y, z) => x + y + z;
add(1, 2, 3); /* 6 */

Para funções com corpo grande, utilize blocos para sua definição:

let greetMore = (name) => {
  let part1 = "Hello";
  part1 ++ " " ++ name
};

Sem Argumento

Uma função sempre recebe um argumento; mas as vezes a utilizamos para, ex. efeitos colaterais, e não temos nada para passar na chamada da função. Em outras linguagens nós conceitualmente "não passamos um argumento". No Reason, todas as função recebem um argumento; nós convencionalmente passamos o valor (), que é chamado de "unit".

/* recebe & destrutura o argumento unit */
let logSomething = () => {
  print_endline("hello");
  print_endline("world")
};

/* chama a função com o valor do tipo unit */
logSomething();

() é um valor totalmente normal, o único valor possível para unit. Reason fornece essa conveniência na sintaxe.

Argumentos Nomeados

Funções com vários argumentos, especialmente aqueles argumentos que são do mesmo tipo, podem ser confusas para chamar.

let addCoordinates = (x, y) => {
  /* usar x e y aqui */
};
...
addCoordinates(5, 6); /* quem é x, quem é y? */

Em OCaml/Reason, você pode atrelar nome para um argumento:

let addCoordinates = (~x, ~y) => {
  /* usar x e y aqui */
};
...
addCoordinates(~x=5, ~y=6);

Já que temos currying (explicado mais abaixo), nós podemos passar os argumentos em qualquer ordem:

addCoordinates(~y=6, ~x=5);

O ~x na declaração significa que a função aceita um argumento nomeado de x e pode se referir a ele no corpo da função pelo mesmo nome. Você também pode se referir aos argumentos dentro do corpo da função por um nome diferente para ser mais preciso:

let drawCircle = (~radius as r, ~color as c) => {
  setColor(c);
  startAt(r, r);
  ...
};

drawCircle(~radius=10, ~color="red");

De fato, (~radius) é apenas um abreviação (chamado de punning) para (~radius as radius).

Aqui está a sintaxe para argumentos tipados:

let drawCircle = (~radius as r: int, ~color as c: string) => ...;

Currying

As funções de Reason podem automaticamente serem parcialmente chamadas:

let add = (x, y) => x + y;
let addFive = add(5);
let eleven = addFive(6);
let twelve = addFive(7);

Na verdade, o código acima add não é nada além de açúcar sintático para isso:

let add = (x) => (y) => x + y;

OCaml otimiza isso para evitar alocação desnecessária de função (2 funções aqui, ingenuamente falando) sempre que possível! Desta forma, nós podemos obter

  • Sintaxe legal
  • Currying de graça (toda função recebe um único argumento, na verdade!)
  • Nenhum custo de perfomance

Argumento Opcional Nomeado

Funções com argumentos nomeados podem ter parâmetros opcionais em sua declaração. Você pode então omiti-los quando chamar a função.

/* argumento radius pode ser omitido */
let drawCircle = (~color, ~radius=?, ()) => {
  setColor(color);
  switch radius {
  | None => startAt(1, 1)
  | Some(r_) => startAt(r_, r_)
  }
};

Quando é dada essa sintaxe, radius é encapsulado com o tipo option, provido pela biblioteca padrão, cujo valor default é None. Se provido, ele será encapsulado com Some. Então, o tipo radius pode ser None ou Some(int) aqui.

Nota: None | Some(foo) é um tipo de estrutura de dado chamado de variante, descrito mais cedo. Especificamente esse variante é provido pela biblioteca padrão. Ele é chamado de option. Sua definição é: type option('a) = None | Some('a).

Note o unit () no final de drawCircle. Sem isso, uma vez que radius and color são ambos nomeados, eles podem ser curried, e podem ser aplicados sem qualquer ordem, desse modo é confuso saber o que isso significa:

let whatIsThis = drawCircle(~color);

O whatIsThis é uma função curried drawCircle esperando um parâmetro opcional radius para ser aplicado? Ou já foi aplicado? Para evitar essa confusão, adiciona-se um argumento posicional (ou seja, um não nomeado) ao drawCircle (convencionalmente ()), e então OCaml irá, como uma regra de ouro, presumir que o argumento opcional nomeado foi provido.

let curriedFunction = drawCircle(~color);
let actualResultWithoutProvidingRadius = drawCircle(~color, ());

Optional Passado Explícitamente

Algumas vezes, você pode querer encaminhar um valor para um função sem saber se o valor é None ou Some(a). Sem pensar muito, você faria:

let result =
  switch payloadRadius {
  | None => drawCircle(~color, ())
  | Some(r) => drawCircle(~color, ~radius=r, ())
  };

Isso fica tedioso rapidamente. Nós disponibilizamos um atalho:

let result = drawCircle(~color, ~radius=?payloadRadius, ());

Isso quer dizer "eu entendo que radius é opcional, e que quando eu passar ele como valor ele precisa ser um int, mas eu não sei se o valor que estou passando é None ou Some(val), então eu passarei o wrapper do option inteiro.

Optional com Valores Padrões

Um argumento nomeado opcional pode também ter um valor padrão. Neste caso, eles não são colocados dentro de um wrapper do tipo option.

let drawCircle = (~radius=1, ~color, ()) => {
  setColor(color);
  startAt(radius, radius)
};

Funções Recursivas

Por padrão, um valor não pode ver o nó para o qual aponta, mas incluindo a palavra-chave rec em um nó let torna isto possível. Isto permite que funções vejam e chamem elas mesmas, dando o poder da recursão.

let rec neverTerminate = () => neverTerminate();

Funções Mutualmente Recursivas

Funções mutuamente recursivas iniciam como uma simples função recursiva usando a palavra-chave rec, e então são encadeadas juntas com and:

let rec callSecond = () => callFirst()
and callFirst = () => callSecond();

Observe que não há nenhum ponto e virgula no fim da primeira linha e nenhum let na segunda.

Dicas & Truques

Dicas para a sintaxe das funções:

Declaração

/ * função anônima. Listado apenas por completude */
(x) => (y) => 1;
/* açúcar sintático para a função acima */
(x, y) => 1;
/* atribuindo um nome */
let add = (x, y) => 1;

/* argumentos nomeados */
let add = (~first as x, ~second as y) => x + y;
/* com açúcar sintático de punning */
let add = (~first, ~second) => first + second;

/* argumentos nomeados com valor padrão */
let add = (~first as x=1, ~second as y=2) => x + y;
/* com punning */
let add = (~first=1, ~second=2) => first + second;

/* argumento opcional */
let add = (~first as x=?, ~second as y=?) => switch x {...};
/* com punning */
let add = (~first=?, ~second=?) => switch first {...};

Com Anotação de Tipo

/* função anônima */
(x: int) => (y: int): int => 1;
/* açúcar sintático da função acima */
(x: int, y: int): int => 1;
/* atribuindo um nome */
let add = (x: int, y: int): int => 1;

/* argumentos nomeados */
let add = (~first as x: int, ~second as y: int) : int => x + y;
/* com punning sugar */
let add = (~first: int, ~second: int) : int => first + second;

/* argumentos nomeados com valor padrão */
let add = (~first as x: int=1, ~second as y: int=2) : int => x + y;
/* com punning sugar */
let add = (~first: int=1, ~second: int=2) : int => first + second;

/* argumento opcional */
let add = (~first as x: option(int)=?, ~second as y: option(int)=?) : int => switch x {...};
/* com punning sugar */
/* note que a chamada irá passar um `int`, não um`option int` */
/* dentro da função, `first` e `second` são `option int`. */
let add = (~first: option(int)=?, ~second: option(int)=?) : int => switch first {...};

Aplicação

/* aplicação anônima. Listado por completude, apenas */
add(x)(y);
/* açúcar sintático para a expressão acima */
add(x, y);

/* argumentos nomeados */
add(~first=1, ~second=2);
/* com punning sugar */
add(~first, ~second);

/* aplicação com um valor padrão. Mesma coisa que uma aplicação comum */
add(~first=1, ~second=2);

/* aplicação opcional explícita */
add(~first=?Some(1), ~second=?Some(2));
/* com punning */
add(~first?, ~second?);

Com Anotação de Tipo

/* aplicação anônima */
add(x: int)(y: int);

/* argumentos nomeados */
add(~first=1: int, ~second=2: int);
/* com punning sugar */
add(~first: int, ~second: int);

/* aplicacação com valor padrão. Mesma coisa que uma aplicação comum */
add(~first=1: int, ~second=2: int);

/* aplicação opcional explícita */
add(~first=?Some(1): option(int), ~second=?Some(2): option(int));
/* com punning sugar */
add(~first: option(int)?, ~second: option(int)?);

Assinatura de Tipo Independente

/* tipo do primeiro argumento, tipo do segundo argumento, retorno do tipo */
type foo = int => int => int;
/* açúcar sintático para o listado acima */
type foo = (int, int) => int;

/* argumentos nomeados */
type foo = (~first: int, ~second: int) => int;

/* argumentos nomeados com valor padrão */
type foo = (~first: int=?, ~second: int=?) => int;

Em arquivos de interface

Anotar uma função pela implementação do arquivo (.re):

let add: int => int => int;
/* açúcar sintático para a função acima */
let add: (int, int) => int;

Se aplica as mesmas regras das seções anteriores, exceto por substituir type foo = bar por let add: bar.

Não confundo isso com realmente exportar um tipo de um arquivo de interface. let add: bar anota um valor existente bar do arquivo de implementação. type foo = bar exporta um tipo de mesma forma do arquivo de implementação.
