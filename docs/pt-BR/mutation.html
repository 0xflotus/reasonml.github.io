Mutação

Reason possui ótimo suporte a programação imperativa tradicional com mutações. Você deve usar essas funcionalidades com moderação, entretanto em algumas ocasiões elas permitem você escrever código com mais performance e em um padrão mais familiar.

Mutação de Let-binding

Let-bindings são imutáveis, mas você pode envolvê-los com ref, que é como uma caixa cujo conteúdo pode ser alterado:

let foo = ref(5);

Uso

Você pode pegar o valor real de um ref através do operador pós-fixo ^:

let five = foo^; /* 5 */

Atribua uma novo valor para foo deste jeito:

foo := 6;

Note que o binding five anterior permanece como 5, dado que este foi definido com o valor "dentro de caixa" ref, não a referência ref por si só.

Dicas & Truques

Brincadeira! ref não é realmente uma funcionalidade extra! É somente açúcar sintático para um tipo de registro mutável chamadoref predefinido na biblioteca padrão (busque por "References" neste link). Segue a versão sem açúcar sintático:

let foo = {contents: 5};
let five = foo.contents;
foo.contents = 6;

Antes de buscar ref, saiba que você pode conseguir "mutações" simples locais através de let bindings que se sobrepõem a bindings anteriores:

let foo = 10;
let foo = someCondition ? foo + 5 : foo;
print_int(foo); /* ou 15 ou 10 */
