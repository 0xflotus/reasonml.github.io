Objeto

Na maior parte do tempo, em Reason, você pode usar record para agrupar valores nomeados. Em algumas situações específicas, entretanto, você talvez possa querer usar algo com funcionalidade similar, chamado object; que são mais flexíveis, mas vêm com alguns trade-offs.

Se você vem de JavaScript, provavelmente você não está procurando objetos Reason planos. Ir para a Dica; Veja abaixo.

Uso

Declaração de tipo

Um objeto não precisa de uma declaração de tipo, embora ele pode ter um. Parece ser um record, exceto com um .:

type tesla = {
  .
  color: string
};

O ponto no início indica que se trata de um tipo de objeto "fechado", significa que um objeto se baseia este tipo deve ter exatamente desta forma.

type car('a) = {
  ..
  color: string
} as 'a;

Dois pontos, também chamados de elisão, indicam que este é um tipo de objeto "aberto" e, portanto, também pode conter outros valores e métodos. Um objeto aberto também é polimórfico e, portanto, requer um parâmetro.

Criação

type tesla = {.
  drive: int => int
};

let obj: tesla = {
  val hasEnvy = ref(false);
  pub drive = (speed) => {
    this#enableEnvy(true);
    speed
  };
  pri enableEnvy = (envy) => hasEnvy := envy
};

Esse objeto é um objeto do tipo tesla e possui um método público chamado de drive. Também contém um método privado chamado de enableEnvy que somente é acessível dentro do contexto do objeto.

Como você pode ver, um objeto Reason pode também acessar this. O comportamento do objeto JavaScriptthis pode ser estranho; O this do Reason sempre aponta para o objeto em si corretamente.

O exemplo a seguir mostra um tipo de objeto aberto que usa um tipo como parâmetro. O parâmetro do tipo do objeto é necessário para implementar todos os métodos do tipo do objeto aberto.

type tesla('a) = {
  ..
  drive: int => int
} as 'a;

let obj: tesla({. drive: int => int, doYouWant: unit => bool}) = {
  val hasEnvy = ref(false);
  pub drive = (speed) => {
    this#enableEnvy(true);
    speed
  };
  pub doYouWant = () => hasEnvy^;
  pri enableEnvy = (envy) => hasEnvy := envy
};

Você pode usar o objeto acima assim:

obj#doYouWant();

Dicas

Se você vem de JavaScript, você provavelmente não procura objetos da versão vanilla do Reason, mas sim do objeto especial do BuckleScript. Estas são diferentes onde:

  • acessa os campos através de ##.
  • se usa #= para a mutação, caso o campo seja mutável (usando bs.set).
  • sempre vem com um Js.t (como um parâmetro tipado para o tipo Js.t).
  • compila para um objeto JS.

De tanto que são usados, O Reason fornece um objeto-valor BS [%bs.obj {foo: bar}] com um syntax sugar especial: {"foo": bar}. Parece basicamente com um quoted record. Da mesma forma para tipos: {. "foo": string}
