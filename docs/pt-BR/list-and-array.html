Lista & Array

Lista

Listas são:

  • homogêneas
  • imutável
  • rápidas ao prepender itens
let myList = [1, 2, 3];

Listas no Reason são simples, são listas encadeadas.

Uso

Você usará lista pela sua capacidade de ser redimensionada, sua rápida operação de prefixar (adicionar um item à "cabeça"), e à sua rápida operação de bipartição, todas garantindo imutabilidade, porém de maneira eficiente!

A biblioteca padrão provê o módulo de Lista (e sua contraparte com argumentos nomeados, ListLabels).

Prefixagem Imutável

Use o operador de expansão, que é apenas List.cons:

let myList = [1, 2, 3];
let anotherList = [0, ...myList];

myList não foi modificado. anotherList agora é [0, 1, 2, 3]. Isto é eficiente (tempo constante, não linear). Os últimos 3 elementos de anotherListsão compartilhados com myList!

Note que [a, ...b, ...c] é um erro de sintaxe. Nós não suportamos múltiplas expansões em uma lista. Isso seria acidentalmente uma operação com tempo linear (O(b)), dado que cada item de b seria adicionado um a um à cabeça de c. Você poder usar List.concat para isso.

Alterar um item arbitrário no meio de uma lista é desencorajado, dado que a performance e gastos extras de alocação seriam lineares (O(n)).

Acesso

switch (descrito na seção de pattern matching) é geralmente usado para acessar itens de listas:

let mensagem =
  switch (minhaLista) {
  | [] =>> "Esta lista está vazia"
  | [a, ...resto] => >"A cabeça da lista é a string " ++ a
  };

Para acessar um item arbitrário de uma lista, use List.nth.

Dicas & Truques

Fique à vontade para alocar quantas listas vazias quanto quiser. Conforme explicado na seção de variantes para listas, uma lista vazia é, na verdade, um construtor de variante sem parâmetro por baixo dos panos, que compila para um mero inteiro. Sem necessitar de alocação extra de memória.

Para entender como prefixar pode ser imutável e O(1) ao mesmo tempo, também confira o link anterior.

Decisões de Design

No futuro, talvez possamos fornecer pronta uma estrutura de dados do tipo lista que seja imutável, redimensionável e tenha operações rápidas para todos os casos.

Array

Arrays são como listas, com exceção de serem:

  • mutáveis
  • rápidos para acesso aleatório & alterações de elementos
  • com tamanho fixo na plataforma nativa (tamanho flexível em JavaScript)

Você deve envolvê-los com [| e |].

let myArray = [|"hello", "world", "how are you"|];

Uso

Módulos Array e ArrayLabel da biblioteca padrão. Para compilação para JS, você também pode usar a habitual API de ligação com Js.Array.

Acesse & altere um item de um array conforme:

let myArray = [|"hello", "world", "how are you"|];

let firstItem = myArray[0]; /* "hello" */

myArray[0] = "hey";

/* agora [|"hey", "world", "how are you"|] */

Os acesso e alteração de item de array acima são somente açúcar sintático para Array.get/Array.set.

Dicas & Truques

Se você está compilando para JavaScript, saiba que arrays em Reason são mapeados diretamente para arrays em JavaScript, e vice-versa. Assim, ainda que arrays tenham tamanho fixo no nativo, você ainda pode usar a API de Js.Array para redimensioná-los. Tá tudo bem.
