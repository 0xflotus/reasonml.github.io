Exemplos para Iniciantes

Um exemplo vale mais que mil palavras.

Essa sessão é dedicada aos iniciantes tentando descobrir os idiomas gerais & convenções no Reason e BuckleScript. Se você é um iniciante que tem uma boa idéia para um exemplo, por favor sugira uma edição!

Usando o tipo option

option é uma variante que vem com a biblioteca padrão. Ela elimina a necessidade de valores nulos presente em outras línguagens.

let possiblyNullValue1 = None;
let possiblyNullValue2: option(string) = Some("Hello@");

switch (possiblyNullValue2) {
| None => print_endline("Nothing to see here.")
| Some(message) => print_endline(message)
};

Criando um tipo parametrizado

type universityStudent = {gpa: float};

type response('studentType) = {
  status: int,
  student: 'studentType
};

let result: response(universityStudent) = fetchDataFromServer();

Criando um Objeto JS

Assumindo que você esteja compilando para JS , é claro.

let obj1 = {
  "name": "John",
  "age": 30
};
/* Compila para um objeto JS da mesma maneira que você está vendo*/

Observe que o código acima não é um registro; as chaves são strings em aspas duplas. Esse é o açúcar sintático do Reason para bs.obj. O tipo é inferido. O próximo exemplifica anotação de tipos explicitamente.

Tipando um Objeto JS

type payload = {.
  "name": string,
  "age": int
};

let obj1: payload = {"name": "John", "age": 30};

Observe que {. name: string, age: int} é a sintaxe para uma declaração de tipo de objeto Reason/OCaml (não um registro!). Ele é elevado para Js.t para que BuckleScript veja o tipo e compile-o corretamente para um objeto JavaScript regular. Normalmente, objetos OCaml não elevados são compilados de forma diferente (raramente é necessário atualmente).

Conectando a um Módulo JS com Export Padrão

Considerando que o módulo chamado store.js, tem um export padrão (default) e um método chamado getDate.

type store = {. "getDate": [@bs.meth] (unit => float)};
[@bs.module] external store : store = "./store";

Js.log(store);
Js.log(store##getDate());

Checando por tipos que podem ser nulos usando option

Para uma função que potencialmente receberá argumentos null ou undefined, é idiomático converter para uma option. A conversão é feita através das funções auxiliares do Bucklescript no módulo do BuckleScript Js.Nullable. Nesse caso, to_opt:

let greetByName = (possiblyNullName) => {
  let optionName = Js.Nullable.to_opt(possiblyNullName);
  switch (optionName) {
  | None => "Hi"
  | Some(name) => "Hello " ++ name
  }
};

Essa verificação compila para possiblyNullName == null em JS, assim ela verifica a presença de null ou undefined.
