External

external, ou interface de função externa (foreign function interface) "FFI", ou simplesmente "interop" (de "interoperability" - interoperabilidade) é como Reason se comunica com outras linguagens, como C ou JavaScript.

Imagine escrever um let binding, porém com o corpo omitido e o tipo explicitamente descrito:

external minhaFuncaoEmC : int => string = "aFuncaoEmC";

[@bs.val] external elementosPorNomeDeClasse : string => array(Dom.element) =
  "document.getElementsByClassName";

(O trecho acima é um external específico de BuckleScript que faz referência a uma função JavaScript de mesmo nome.)

Nota: externals só pode estar no escopo superior ou dentro de uma definição de módulo. Você não pode declará-lo em um corpo de função, por exemplo.

Uso

Você irá usar um valor/função external como se o mesmo fosse um let binding normal.

Dicas

Se você vem de um background JavaScript: invista algum tempo para aprender externals de BuckleScript! No início você provavelmente esbarrará em uma porção de externals antes de escrever código Reason idiomático 100% puro.

Decisões de design

Reason leva muito a sério a questão de interoperabilidade com código existente. Nosso sistema de tipos possui garantias muito fortes. Entretanto, essa característica forte também significa que, sem um bom sistema de interoperabilidade, seria muito difícil converter gradualmente uma base de código para Reason. Fortunately, the FFI allows us to cooperate very well with dirty, existing code. A combinação de um sistema de tipos correto (aka 'sound') com boa interoperabilidade significa que obtemos os benefícios de um sistema com tipagem gradual tradicional no que diz respeito à cobertura incremental por tipos na conversão de uma base de código, sem a desvantagem de de tal sistema com tipagem gradual: funcionalidades complexas para dar suporte a padrões existentes, análise lenta, rendimentos decrescentes em termos de cobertura por tipos etc.
