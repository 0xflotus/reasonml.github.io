Miscelânea

Extensão para navegador: Reason-tools

Reason-tools te permite alternar entre a sintaxe de OCaml e a de Reason quando estiver buscando tutoriais e documentações escritas nessas sintaxes.

BuckleScript

Você verá BuckleScript extensivamente ao longo do restante desta documentação, uma vez que ele é o motor que alimenta a nossa compilação JS. Seus binários globais são instalados através do npm install -g bs-platform.

Outras Ferramentas

A instalação global que você fez na sessão passada vem com algumas ferramentas úteis, descritas aqui.

Refmt

refmt ("Reason format") é o binário que possibilita o recurso de auto-formatar de nossos editores. Ele recebe seu código e cospe ele formatado bonitinho. Ele também serve para converter a sintaxe de/para Reason/OCaml. A comunidade Reason usa ele para alcançar um estilo de código consistente ao longo de diferentes projetos, e para evitar consumir tempo em formatação manual e debates estilísticos.

It responsively formats the code based on the editor width. In other words, it doesn't just naively break to the next line at a certain characters limit; it solves the layout constraints and arranges your code accordingly.

refmt pode ser usado diretamente no terminal. Por exemplo, para formatar seu código fora do editor, faça refmt --in-place meuArquivo.re. Veja refmt --help para todas as opções.

Merlin

Merlin é o motor subjacente que possibilita as dicas de tipos, refatoração, erros em tempo real, ir para definições, etc. para os nossos editores. Sua ferramenta de linha de comando chama-se ocamlmerlin, embora você provavelmente não vá chamá-la manualmente (os editores a iniciam e fazem requisições para ela).

Para configurar o Merlin para entender o seu projeto, você deve criar um arquivo .merlin na raiz (documentação aqui). Para o workflow JS, essa configuração é gerada automaticamente para você pelo bsb do BuckleScript.

REPL

Reason vem com um REPL chamado rtop, o qual, quando chamado, te permite executar código interativamente. Também possui autocompletar inteligente baseado nos tipos.

Use #show <symbol>; to inspect the environment, #toggle_syntax; to switch between syntax and #quit; to close your REPL session.

Observe que o rtop atualmente não funciona facilmente com pacotes e externals. Nós recomendamos executar o código dentro do nosso parquinho de diversões aqui.

re:bench

re:bench é um ambiente online para benchmarking. Ele suporta casos de teste escritos em Reason, OCaml e JavaScript, e permite o compartilhamento de benchmarks através de URLs únicas. É compilado para JS usando BuckleScript.

Redex

Redex: the Reason packages registry.

ocamlc, ocamlopt, ocamlrun

ocamlc and ocamlopt are the bare ocaml compilers.
