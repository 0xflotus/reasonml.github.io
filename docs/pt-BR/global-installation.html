Instalação Global

Antes de configurar os plugins do editor, você precisa instalar globalmente os binários necessários.

(Recomendado) Através do Npm/Yarn

PlataformaComando de Instalação
MacOSnpm install -g reason-cli@3.2.0-darwin
Linuxnpm install -g reason-cli@3.2.0-linux
WindowsPor favor, veja https://github.com/reasonml/reasonml.github.io/issues/195

Nota: O download é bem grande (aproximadamente 100MB), porém o npm não irá mostrar o progresso enquanto baixa.

(Alternativa) Através do OPAM

OPAM é o gerenciador de pacotes nativo do OCaml. Se você vem de OCaml e não possui o npm/yarn, você pode instalar opcionalmente desta forma, mas cuidado!

Se você está no Windows, consulte https://github.com/reasonml/reasonml.github.io/issues/195.

opam update
eval $(opam config env)
opam install reason.3.0.4 merlin.2.5.4

Resolução de Problemas

Erro de Instalação

Se sua instalação falhar, talvez seja porque você está usando o npm 5.4.0 (npm --version). Havia um bug conhecido no npm que foi arrumado no 5.4.2. Atualize o npm e as coisas deveram funcionar.

Se a solução acima falhar, tente https://github.com/reasonml/reasonml.github.io/pull/157. Se funcionar, por favor, vote nesta issue. Não temos certeza de que esta seja a solução adequada.

E se as coisas ainda não funcionam, por favor, envie uma issue em https://github.com/reasonml/reason-cli/issues. Desculpe o incómodo.

O Plugin do Editor não está funcionando

  • Se você estiver no Windows, o suporte atual para a ferramentas de edição é instável. Por favor nos ajude a melhorá-lo em https://github.com/reasonml/reasonml.github.io/issues/195. Obrigado!
  • Certifique-se de reiniciar o seu editor. Alguns deles podem não identificar seu novo ambiente shell (que agora inclui os novos binários instalados) sem reiniciar.
  • Tente o seguinte: ```readlink which ocamlmerlin refmt ocamlmerlin-reason```` Isso deveria mostrar 3 caminhos que contenham a palavrareason-clise oreason-cli` for instalado devidamente.
  • Verifique a versão do Merlin: ocamlmerlin -version Isso deveria dizer "The Merlin toolkit version 2.5.x, for OCaml 4.02.3". E não OCaml 4.03, e nem 4.04, etc.

Mensagem de Erro do Editor: Unbound Module Js, Etc.

Certifique-se você tenha construído o seu projeto pelo menos uma vez; o diagnóstico não inicia até que isso ocorra. O comando de construção varia por projeto, mas é frequentemente npm run build (que normalmente chama bsb -make-world internamente).

Se você está no Visual Studio Code, certifique-se que você tenha aberto o editor na raiz do projeto (onde package.json e bsconfig.json estão). Você pode fazê-lo, por exemplo, invocando code . no terminal na raiz.
