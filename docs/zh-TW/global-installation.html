全域安裝

在設定編輯器外掛之前，你需要先安裝全域二進制檔，外掛需要用到。

（推薦）透過 Npm/Yarn

平台安裝指令
macOSnpm install -g https://github.com/reasonml/reason-cli/archive/3.0.4-bin-darwin.tar.gz
Linuxnpm install -g https://github.com/reasonml/reason-cli/archive/3.0.4-bin-linux.tar.gz
Windows請見 https://github.com/reasonml/reasonml.github.io/issues/195

注意：目前的綑綁十分龐大（約 100MB），下載時 npm 並不會顯示進度條。

（替代）透過 OPAM

OPAM 係 OCaml 原生的套件管理程式。如果你來自 OCaml 沒有 npm/yarn，你可選擇性採用這個方式，但請小心使用！

如果使用 Windows，請見https://github.com/reasonml/reasonml.github.io/issues/195.

opam update
opam switch 4.02.3 # 必要！
eval $(opam config env)
opam install reason.3.0.4 merlin.2.5.4

疑難排解

安裝失敗

如果安裝失敗，可能原因是你使用 npm 5.4.0（npm --version）。這是個已知的臭蟲且已經在 5.4.2 修正。升級 npm 之後應能正常安裝。

如果還是失敗，請試試 https://github.com/reasonml/reasonml.github.io/pull/157。如果這能解決你的問題，請在該頁面內投下贊成票。目前不確定這是個合適的修正。

最後，如果仍然失敗，請在 https://github.com/reasonml/reason-cli/issues 發起新問題。抱歉造成您的困擾。

編輯器外掛沒有作用

  • 如果使用 Windows，目前編輯器工具鍊尚未穩定。請協助我們改善 https://github.com/reasonml/reasonml.github.io/issues/195。謝謝！
  • 確認重新啟動編輯器。某些無法取得新的 shell 環境（包含了新安裝的二進制檔）。
  • 試試下列指令： ```readlink which ocamlmerlin refmt ocamlmerlin-reason```` 這會出現三個路徑包含這些字串reason-cli如果reason-cli` 安裝成功。
  • 檢查 Merlin 版本： ocamlmerlin -version 這會顯示 “The Merlin toolkit version 2.5.x, for Ocaml 4.02.3”。不是 OCaml 4.03，也不是 4.04，等等。

編輯器錯誤訊息：Unbound Module Js，等等。

確認專案至少編譯過一次；診斷無法進行直到編譯過一次。 不同專案的建置指令不同，但通常是 npm run build（該指令通常會呼叫 bsb -make-world）。

如果使用 Visual Studio Code，確認是在專案的根目錄（即 package.json 及 bsconfig.json 的所在處）下開啟編輯器。 你可以，例如，透過終端機在專案目錄下執行 code .。
