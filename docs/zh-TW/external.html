外部

external，或 "FFI"（外來函式界面），或簡單地 "交互"（ 意思是 "交互作用性"）是 Reason 如何與其他語言進行溝通，像是 C 或 JavaScript。

想像成撰寫 let 繫結，但省略其內容並將其型別明確地寫下：

external myCFunction : int => string = "theCFunction";

[@bs.val] external getElementsByClassName : string => array(Dom.element) =
  "document.getElementsByClassName";

（上述為 BuckleScript-特定的 external 用來綁定 JavaScript 函式，繫結名稱與欲綁定函式名稱相同。）

注意：external 只能在最上層使用，或者在模組定義內。無法在如函式內部中宣告。

用法

使用外來值/函式就如同使用 let 繫結。

提示 & 技巧

如果你是來自 JavaScript 背景：花些時間了解 BuckleScript externals！ 剛開始時，在你撰寫 100% 純慣用 Reason 程式碼之前，可能會先遇到許多 external 。

設計決策

Reason 非常重視與既有程式碼的交互作用。 型別系統有很強的保證。 然而，這種強大的功能也表示，少了良好的交互系統，將會非常難以漸進轉換既有程式碼到 Reason。 幸虧，FFI 讓我們能夠與骯髒，既有的程式碼有非常好的合作。 強大的型別系統 + 良好交互作用的組合意味著我們注視增量程式庫覆蓋率 & 轉換以獲得傳統漸增型別系統的益處，免除漸增型別系統的缺點：複雜功能以支援既有樣式，慢速分析，就型別涵蓋率減少回報，等。
