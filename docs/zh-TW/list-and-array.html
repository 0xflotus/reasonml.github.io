List & 陣列

List

Lists 是：

  • 同質
  • 不可變（immutable）
  • 前插十分快速
let myList = [1, 2, 3];

Reason lists 是個簡單、單獨地鏈結串列（linked list）。

用法

為了長度伸縮性，你可能會使用 list，快速前插（插入最前面），及快速分割，這些都是不可變的有效率！

標準函式庫提供 List 模組（以及標籤化的對應模組，ListLabels）。

不可變插入首元素

使用擴散語法，就是 List.cons：

let myList = [1, 2, 3];
let anotherList = [0, ...myList];

myList 不會改變。 anotherList 現在是 [0, 1, 2, 3]。 這個十分有效率（常數時間，非線性）。 anotherList 的最後 3 元素與 myList 共享！

注意 [a, ...b, ...c] 是錯誤語法。 我們不支援同一個 list 有多個擴散。 這會造成意外地線性操作（O(b)），由於 b 的每個元素將會逐一加在 c 的前方。 你可以使用 List.concat。

不鼓勵更新 lists 的任意中間項目，因為效率及配置成本將會是線性（O(n)）。

存取

switch（樣式匹配章節介紹）通常用來存取 list 項目：

let message =
  switch (myList) {
  | [] => "這個 list 是空的"
  | [a, ...rest] => "list 的開頭是字串 " ++ string_of_int(a)
  };

若要存取任意 list item，使用List.nth。

提示 & 技巧

你可以隨意配置空的 list。 如同變異章節的 list 說明，空的 list 實際上只是個無參數的變異建構子，只會編譯成整數。 不需要額外配置記憶體。

若要了解如何同時前插不可變且 O(1)，請參考上一個連結。

設計決策

未來，我們可能會提供 list 資料結構，同時具有不可變、伸縮長度及所有快速操作的功能。

陣列

陣列像是 lists，但是是：

  • 可變的
  • 快速的隨機存取 & 修改
  • 原生固定長度（JavaScript 彈性長度）

使用 [| 及 |] 圍繞。

let myArray = [|"hello", "world", "how are you"|];

用法

標準函式庫 Array 及 ArrayLabel 模組。 若要編譯 JS，這裡有熟悉的 Js.Array API 綁定。

存取 & 修改陣列就像是：

let myArray = [|"hello", "world", "how are you"|];

let firstItem = myArray[0]; /* "hello" */

myArray[0] = "hey";

/* 現在 [|"hey", "world", "how are you"|] */

上述陣列存取/修改只是 Array.get/Array.set 的語法糖。

提示 & 技巧

如果編譯 JavaScript，Reason 陣列直接地對應 JavaScript 陣列，反之亦然。 因此，儘管在原生陣列是固定長度，仍然可以使用 Js.Array API 調整長度。 這沒問題。
