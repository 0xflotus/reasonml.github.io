布林值

布林的型別為 bool，可以是 true 或者 false。常見操作有：

  • &&：邏輯 and
  • ||：邏輯 or
  • !：邏輯 not
  • <=、>=、<、>
  • ==: structural equal, compares data structures deeply: (1, 2) == (1, 2) is true. Convenient, but use with caution
  • ===：參照相等，淺的比對。 (1, 2) === (1, 2) 為 false。let myTuple = (1, 2); myTuple === myTuple 為 true。
  • !=: structural unequal
  • !==：參照不相等

用法

**注意：JS true 和 false 與 Reason/BuckleScript true 和 false 不一樣！ 請不要交互地使用，如果沒有適時的使用 BuckleScript 的 Js.to_bool 和 Js.Boolean.to_js_boolean 進行轉換。

提示 & 技巧

Use structural equal tastefully. 這很方便，但可能會意外地讓你比對兩個深層巢狀的資料結構，導致效能低落。 並不是每次都很清楚了解何謂 "相等"。 例如，一塊資料 foo 是否相等緩慢 foo？ 理想上，這是可抽換的。 即將到來的改變將會讓此成真並且可靠；如果有興趣，請見隱含模組化。

設計決策

*此段落假設已對變異有了解。如果你是第一次閱讀指南，歡迎稍後再來 *！

布林是（只在概念上）一種變異的特殊案例：type bool = True | False。 設計上，優雅地省去在型別系統裡撰寫布林型別程式碼。 缺點是，就如同變異，建構子會編譯比較不容易閱讀但是更快速的表達。 這就是為什麼 BuckleScript 會缺乏訊息，在後者階段，成功將 Reason 的 true/false 編譯成 JavaScript 的 true/false。
