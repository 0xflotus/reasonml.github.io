瀏覽器擴充：Reason-tools

Reason-tools 讓你可以瀏覽 OCaml 或 Reason 的教學或文件時，快速切換 OCaml 與 Reason 語法。

BuckleScript

接下來的內容你會廣泛地看到 BuckleScript 出現，這是個用來編譯 JS 的引擎。 透過指令 npm install -g bs-platform 進行全域的二進制檔安裝。

其他工具

前一章節所安裝的全域工具，伴隨一些額外有用的工具，以下簡介：

Refmt

refmt（“Reason format”）是個二進制檔，用來達成編輯器自動縮排程式碼功能。 輸入原始程式碼並輸出良好格式化的程式碼。 同時轉換 Reason/OCaml 兩者語法。 Reason 社群用此達到不同專案間一致的程式碼風格，省去手動調整的時間 & 風格上的辯論。

根據編輯器的寬度，適應性的格式化原始碼。 換句話說，不只是單純地在特定字元長度下換行而已；解決版面配置（layout）限制並根據版面來安排你的程式碼。

refmt 能夠直接在終端機下使用。 例如，格式化不存在編輯器內的程式碼，執行 refmt --in-place myFile.re。 請見 refmt --help 所有選項。

Merlin

Merlin 是個底層引擎驅動強大的型別提示、重構、即時錯誤、跳至定義，等等。 提供給編輯器使用。 其命令列的名稱為 ocamlmerlin，通常不需要手動呼叫（編輯器自行啟動並查詢）。

若要在專案中設定 Merlin，需要在專案根目錄下撰寫 .merlin 檔案（文件請見此） 若是 JS 工作流程，BuckleScript 的 bsb 會自動為您產生。

REPL

Reason 內建 REPL 稱為 rtop，一但執行，便可交互式評估程式碼。特色是智慧、型別驅動的自動補完。

使用 #quit; 關閉 REPL 會期。

注意目前 rtop 中使用套件及 externals 並不容易，建議試試遊樂場來評估程式碼。

re:bench

re:bench 是個線上評效（benchmarking）遊樂場。 支援 Reason、OCaml 及 JavaScript 所撰寫的測試案例，透過獨有的網址進行坪效分享。 使用 BuckleScript 編譯成 JS。

Redex

Redex：Reason 套件中心。

ocamlc、ocamlopt、ocamlrun、rebuild

ocamlc 及 ocamlopt 兩者為 OCaml 編譯器。
