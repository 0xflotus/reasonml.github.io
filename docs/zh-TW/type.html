型別！

型別是 Reason 的亮點！這裡，你會篤見為何許多人如此為此感到興奮。

此章節簡短介紹型別句法，讓你可以閱讀接下來的章節內容而不會感到疑惑。 更多進階主題在更多型別章節。

註記

let 繫結不需要撰寫任何型別：

let score = 10;

Reason 知道 score 是個 int 型別，透過其值 10 判斷出來。這個稱之為 推斷（inference）。

當然型別也可以明確地寫下來：

let score: int = 10;

你也可以在圓括弧中撰寫表達式並且註記型別：

let myInt = 5;
let myInt: int = 5;
let myInt = (5: int) + (4: int);
let add = (x: int, y: int) : int => x + y;
let drawCircle = (~radius as r: int) : unit => ...;

注意：最後一行， (~radius as r: int) 是一種標籤式引數（labeled argument）。詳情請見這裡。

別名

你可以用不同的名稱來參照型別。兩者會相等：

type scoreType = int;
let x: scoreType = 10;

設計決策

Reason 底層是 OCaml，其型別系統已歷數十年工程。下列幾個重要特點：

  • 型別能夠推論出來。 如果沒有手動寫下型別，系統會為你推演型別。 加速原型開發階段。 另外，編輯器的功能，如VSCode 的 所codelens，當撰寫程式碼時，顯示所有型別。

  • 型別涵蓋率永遠 100%。不需要 "型別涵蓋率" 工具！任何一段 Reason 程式碼都有型別。

  • 型別系統完全地"可靠". 這表示，只要程式碼能夠正常編譯，任何型別都不會對你撒謊。 在傳統的、盡力的型別系統中，當型別說"這是整數（integer）不會是 null"，不表示真的不會是 null。 反觀，純 Reason 程式不存在 null 所犯的錯誤。
