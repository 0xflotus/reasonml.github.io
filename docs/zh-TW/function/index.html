函式

最底部有完整函式語法的快捷手冊

你相信嗎？直到現在我們都還沒涵蓋到函式。

函式宣告需要箭頭及傳回表達式。

let greet = (name) => "Hello " ++ name;

這個宣告一個函式，指派給 greet，呼叫看起來像是：

greet("world!"); /* "Hello world!" */

多個參數使用逗點分隔：

let add = (x, y, z) => x + y + z;
add(1, 2, 3); /* 6 */

較長的函式，可以用區域來包住內容：

let greetMore = (name) => {
  let part1 = "Hello";
  part1 ++ " " ++ name
};

無參數

函式必須帶參數；但有時候會用在如副作用（side-effects），且完全不會用到傳進來的東西。 其他語言，概念上傳遞 "無參數"。 Reason，每個函式都要參數；這裡習慣上會傳入 () 值，稱為 "unit"。

/* 接收 & 解構 unit 參數 */
let logSomething = () => {
  print_endline("hello");
  print_endline("world")
};

/* 呼叫函式並傳入 unit 型別的值 */
logSomething();

() 是完全正常的值，unit 的唯一可能值。為了方便 Reason 給予一個特別的語法。

標籤的參數

多個參數的函式，特別是有相同型別的參數，呼叫的時候容易搞混。

let addCoordinates = (x, y) => {
  /* 這裡用到 x 跟 y */
};
/* ... */
addCoordinates(5, 6); /* 哪個是 x，哪個是 y？ */

In OCaml/Reason, you can attach labels to an argument by prefixing the name with the ~ symbol:

let addCoordinates = (~x, ~y) => {
  /* 這裡用到 x 跟 y */
};
/* ... */
addCoordinates(~x=5, ~y=6);

由於 currying （稍後會作說明），可以提供參數於任何順序：

addCoordinates(~y=6, ~x=5);

宣告的 ~x 部份表示函式接受參數帶有標籤 x 且函式內部可以用相同名稱取得。 為了簡潔，也可以用在函式內部使用不同的名稱：

let drawCircle = (~radius as r, ~color as c) => {
  setColor(c);
  startAt(r, r);
  /* ... */
};

drawCircle(~radius=10, ~color="red");

事實上，(~radius) 只是 (~radius as radius) 的速寫法（稱之為 雙關（punning））.

以下為參數撰寫型別的語法：

let drawCircle = (~radius as r: int, ~color as c: string) => ...;

Currying

Reason 函式自動變成部份地呼叫：

let add = (x, y) => x + y;
let addFive = add(5);
let eleven = addFive(6);
let twelve = addFive(7);

實際上，上述 add 只是語法糖：

let add = (x) => (y) => x + y;

OCaml 會盡量做最佳化以避免不必要的函式分配（這裡有 2 個函式，直覺上來說）！ 這樣，得到

  • 更好的語法
  • 自動 currying（事實上，每個函式只有一個參數！）
  • 沒有效能成本

選項的標籤式參數

標籤式函式參數可以宣告為選項。呼叫函式時可以省略。

/* radius 可以忽略 */
let drawCircle = (~color, ~radius=?, ()) => {
  setColor(color);
  switch (radius) {
  | None => startAt(1, 1)
  | Some(r_) => startAt(r_, r_)
  }
};

當使用此語法，radius 是包裝成標準函式庫的 option 型別，預設值是 None。 如果有呼叫方有提供，則會包裝成 Some。 所以這裡 radius 型別值會是 None 或 Some(int)。

注意：None | Some(foo) 是一種稱為變異的資料結構，稍早有說明。 此特殊變異型別係由標準函式庫所提供。 稱之為 option。 定義：type option('a) = None | Some('a)。

注意 drawCircle 最後面的 unit ()。 若少了這個，由於 radius 和 color 都是標籤式，可以 curried，可以不照順序提供，下列情況變得意義不明：

let whatIsThis = drawCircle(~color);

whatIsThis 是要 curried drawCircle 函式，等待選項的 radius 參數？ 還是已經結束呼叫了？ 為了解決這問題，drawCircle 附加一個位置上（又稱非標籤式）參數（方便上使用 ()），而 OCaml 將會，以經驗法則來看，當位置上的參數已經提供，便可假設省略了選項的參數。

let curriedFunction = drawCircle(~color);
let actualResultWithoutProvidingRadius = drawCircle(~color, ());

明確地傳遞選項

有些時候，你會希望在不需要知道某個值是 None 或 Some(a) 直接轉發到函式，天真地，可能會：

let result =
  switch (payloadRadius) {
  | None => drawCircle(~color, ())
  | Some(r) => drawCircle(~color, ~radius=r, ())
  };

很快地就會感到乏味。我們提供捷徑：

let result = drawCircle(~color, ~radius=?payloadRadius, ());

意思是 "我知道 radius 是選項的，當我傳遞進來時必須是 int，然而我不知道傳遞進來的值是 None 或 Some(val)，所以我把整個 option 包裝傳給你"。

選項帶有預設值

選項標籤的參數也可以有預設值。這個情況下，不需要包裝成 option 型別。

let drawCircle = (~radius=1, ~color, ()) => {
  setColor(color);
  startAt(radius, radius)
};

遞迴函式

預設情況，值本身是無法看到綁定它的東西，但只要在 let 加上 rec 關鍵字，就可以看得到。 這允許函式可以看到並且呼叫自己，賦予我們遞迴的能力。

let rec neverTerminate = () => neverTerminate();

互助地遞迴函式

互助地遞迴函式像是使用 rec 關鍵字的單一遞迴函式，但是用 and 串鏈在一起：

let rec callSecond = () => callFirst()
and callFirst = () => callSecond();

注意 第一行結尾沒有分號，第二行沒有 let。

提示 & 技巧

函式語法的快捷手冊:

宣告

/* 匿名函數。 為了完整性而列出 */
(x) => (y) => 1;
/* 上述的語法糖 */
(x, y) => 1;
/* 指派特定名稱 */
let add = (x, y) => 1;

/* 標籤式 */
let add = (~first as x, ~second as y) => x + y;
/* 使用雙關語法糖 */
let add = (~first, ~second) => first + second;

/* 帶有預設值的標籤式 */
let add = (~first as x=1, ~second as y=2) => x + y;
/* 使用雙關 */
let add = (~first=1, ~second=2) => first + second;

/* 選項 */
let add = (~first as x=?, ~second as y=?) => switch (x) {...};
/* 使用雙關 */
let add = (~first=?, ~second=?) => switch (first) {...};

使用型別標註

/* 匿名函式 */
(x: int) => (y: int): int => 1;
/* 上述的語法糖*/
(x: int, y: int): int => 1;
/* 指派特定名稱 */
let add = (x: int, y: int): int => 1;

/* 標籤式 */
let add = (~first as x: int, ~second as y: int) : int => x + y;
/* 使用雙關語法糖 */
let add = (~first: int, ~second: int) : int => first + second;

/* 帶有預設值的標籤式 */
let add = (~first as x: int=1, ~second as y: int=2) : int => x + y;
/* 使用雙關 */
let add = (~first: int=1, ~second: int=2) : int => first + second;

/* 選項 */
let add = (~first as x: option(int)=?, ~second as y: option(int)=?) : int => switch (x) {...};
/* 使用雙關 */
/* 注意，呼叫端傳遞 `int`，而非 `option int` */
/* 函式內部 `first` 和`second` 都是 `option int`。 */
let add = (~first: option(int)=?, ~second: option(int)=?) : int => switch (first) {...};

應用（Application）

/* 匿名函式。 為了完整性而列出 */
add(x)(y);
/* 上述的語法糖 */
add(x, y);

/* 標籤式 */
add(~first=1, ~second=2);
/* 使用雙關語法糖 */
add(~first, ~second);

/* 應用帶預設值 就像一般應用 */
add(~first=1, ~second=2);

/* 明確選項應用 */
add(~first=?Some(1), ~second=?Some(2));
/* 使用雙關 */
add(~first?, ~second?);

使用型別標註

/* 匿名函式 */
add(x: int)(y: int);

/* 標籤式 */
add(~first=1: int, ~second=2: int);
/* 使用雙關語法糖 */
add(~first: int, ~second: int);

/* 應用帶預設值 就像一般應用 */
add(~first=1: int, ~second=2: int);

/* 明確選項應用 */
add(~first=?Some(1): option(int), ~second=?Some(2): option(int));
/* 使用雙關語法糖 */
add(~first: option(int)?, ~second: option(int)?);

單獨型別署名（Signature）

/* first arg type, second arg type, return type */
type foo = int => int => int;
/* sugar for the above */
type foo = (int, int) => int;

/* labeled */
type foo = (~first: int, ~second: int) => int;

/* labeled */
type foo = (~first: int=?, ~second: int=?, unit) => int;

在界面檔中

標註實作檔（.re）的函式：

let add: int => int => int;
/* 上述的語法糖 */
let add: (int, int) => int;

跟前章節一樣規則，除了替換 type foo = bar 成 let add: bar。

不要跟界面檔中實際輸出型別搞混。 let add: bar 標註實作檔的既有值 bar。 type foo = bar 輸出與實作檔相同外型的型別。
