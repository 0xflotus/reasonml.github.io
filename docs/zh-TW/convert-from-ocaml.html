Converting from OCaml

由於 Reason 只是 OCaml 的另一個語法，轉換 OCaml 專案是率直且不需要改變語意。 然而，仍有幾個建置設定及要求修改一些雜項。

OCamlBuild -> Rebuild

Reason 帶來 rebuild 取代 ocamlbuild，這會自動建置任何 Reason 及 OCaml 檔案，不需要額外設定。 允許你一點一滴的在現有 OCaml 專案加入 Reason 檔案。 無論何處用到 ocamlbuild，都換成 rebuild。

Makefile

如果建置系統執行明確的建置指令， 最簡單在 ocamlopt/ocamlc 使用 Reason 的方式是加入下列旗標於編譯步驟中：

# intf-suffix 告訴編譯器從哪裡找到相對應的介面檔案
ocamlopt -pp refmt -intf-suffix rei -impl myFile.re
ocamlopt -pp refmt -intf myFile.rei

如果使用 ocamlbuild 而不使用 rebuild，加入下列指令到 _tags 檔，但這可能還不夠，ocamlc/ocamlopt 會需要 -intf/-impl/-intf-suffix 旗標：

<**/*.{re,.rei}>: package(reason), syntax(utf8)

修正建構子語法

轉換後的 Reason 程式碼可能在變異建構子加上 [@implicit_arity]，像是：C 1 2 [@implicit_arity]。 這是因為 OCaml 的模糊語法，期望多個參數的建構子使用 tuple 形式。 所以在剖析期間並不知道 OCaml 的 C (1, 2) 應該翻譯成 Reason 的 C (1, 2) 還是 C 1 2。 預設，這會轉換成 C 1 2 [@implicit_arity]，告訴編譯器可能是 C 1 2 或 C (1, 2)。

若要避免產生 [@implicit_arity]，一種方式是加入 --assume-explicit-arity 到 refmt。 強迫翻譯器產生 C 1 2 而非 C 1 2 [@implicit_arity]。

然而，由於 C 1 2 要求多個參數，如果實際上建構子是接受一個 tuple 參數（例如 Some）時，編譯可能會失敗。 我們已經有一些內部例外規則來涵蓋常見的一個 tuple 參數建構子，這些可以被正確地轉換（例如 Some (1, 2) 將會轉換成 Some (1, 2)而不是無法編譯的 Some 1 2）。

若要提供自己的例外清單，在專案中新增一個檔案，列出預期是單一 tuple 的建構子，一行一個建構子（不需要前綴模組名稱），接著使用 --heuristics-file <filename> 來告訴 refmt 這個檔案內所列出的建構子都視接受單一 tuple 的建構子。

> cat heuristics.txt
  TupleConstructor
  And
  Or

> cat test.ml

type tm =
  TupleConstructor of (int * int)
| MultiArgumentsConstructor of int * int
let x = TupleConstructor(1, 2)
let y = MultiArgumentsConstructor(1, 2)
module Test = struct
  type a = | And of (int * int) | Or of (int * int)
end;;
let a = Test.And (1, 2)
let b = Test.Or (1, 2)
let c = Some (1, 2)

只有列在檔案裡面的建構子識別才會假設接受 tuple 而非多個參數。

> refmt --heuristics-file \
    ./heuristics.txt --assume-explicit-arity \
    --parse ml --print re test.ml

type tm =
  | TupleConstructor((int, int))
  | MultiArgumentsConstructor(int, int);

let x = TupleConstructor((1, 2));

let y = MultiArgumentsConstructor(1, 2);

module Test = {
  type a =
    | And((int, int))
    | Or((int, int));
};

let a = Test.And((1, 2));

let b = Test.Or((1, 2));

let c = Some((1, 2));
