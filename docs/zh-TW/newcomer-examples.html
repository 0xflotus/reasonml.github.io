使用 option 型別

option 是個變體（variant）內建於標準函式庫。排除像是其他語言對於 null 值的需求。

let possiblyNullValue1 = None;
let possiblyNullValue2: option(string) = Some("哈嘍@");

switch (possiblyNullValue2) {
| None => print_endline("沒有東西")
| Some(message) => print_endline(message)
};

新增參數化型別

type universityStudent = {gpa: float};

type response('studentType) = {
  status: int,
  student: 'studentType
};

let result: response(universityStudent) = fetchDataFromServer();

新增 JS 物件

當然，這是假設你要編譯成 JS。

[@bs.deriving abstract]
type payload = {
  name: string,
  age: int
};

let obj1 = payload(~name="John", ~age=30);

/* Compiles to a JS object with the above fields */

Note that the above isn't a record; this is a special BuckleScript feature.

Checking for JS nullable types using the option type

For a function whose argument is passed a JavaScript value that's potentially null or undefined, it's idiomatic to convert it to a Reason option. The conversion is done through the helper functions in Bucklescript's Js.Nullable module. In this case, toOption:

let greetByName = (possiblyNullName) => {
  let optionName = Js.Nullable.toOption(possiblyNullName);
  switch (optionName) {
  | None => "Hi"
  | Some(name) => "Hello " ++ name
  }
};

This check compiles to possiblyNullName == null in JS, so checks for the presence of null or undefined.
