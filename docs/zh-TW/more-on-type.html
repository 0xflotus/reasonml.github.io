更多有關型別

型別參數！

型別可以接受參數，類似其他語言的泛型（generics）。 就像型別是個函式接受參數傳回新的型別！ 參數必須使用 ' 開頭。

參數化型別的用途在於減少重複。使用前：

/* 這是 3 個項目的 tuple，稍後解釋 */
type intCoordinates = (int, int, int);
type floatCoordinates = (float, float, float);

let buddy: intCoordinates = (10, 20, 20);

使用後：

type coordinates('a) = ('a, 'a, 'a);

/* 套用同等的 "型別函式" 傳回 (int, int, int) 型別 */
type intCoordinatesAlias = coordinates(int);

let buddy: intCoordinatesAlias = (10, 20, 20);

/* 或者，更常見的，撰寫成內嵌（inline） */
let buddy: coordinates(float) = (10.5, 20.5, 20.5);

實際上，型別會幫你推論。所以上述更簡潔的版本會是：

let buddy = (10, 20, 20);

型別系統推論它是 (int, int, int) 型別。不需要額外撰寫。

型別參數出現在任何地方。

/* 推論出 `list(string)` */
let greetings = ["hello", "world", "how are you"];

如果型別不接受參數（例如，沒有 "型別函式"），標準函式庫就會需要定義 listOfString 、listOfInt、listOfTuplesOfInt 等型別。

型別可以接受多個參數，並且組合在一起。

type result('a, 'b) =
  | Ok('a)
  | Error('b);

type myPayload = {data: string};

type myPayloadResults('errorType) = list(result(myPayload, 'errorType));

let payloadResults: myPayloadResults(string) = [
  Ok({data: "hi"}),
  Ok({data: "bye"}),
  Error("Something wrong happened!")
];

相互遞迴型別

就像是函式，型別可以用 and 相互遞迴：

type student = {taughtBy: teacher}
and teacher = {students: list(student)};

注意第一行結尾沒有分號，第二行沒有 type。

設計決策

型別系統接受型別參數基本上允許型別階層（type-level）函式。 list(int) 實際是 list 型別函式，接受 int 型別，傳回會拿來使用的最終、具體型別。 你可能注意到其他語言，或多或少稱這個為 "泛型（generics）"。 例如：Java 的 ArrayList<Integer>。

當你使著 "完成事情" 時，套用 The principle of least power。 如果問題領域允許，一定選擇最不抽象（又稱，最具體）解決方法，這樣解決方法才會達到快速和更少需要跨越不穩定的方向。 例如，偏好型別更勝於自由格式的資料，偏好資料導向配置更勝於完全圖靈函式呼叫，偏好函式呼叫更勝於巨集，偏好巨集更勝於專案分叉等。 當你限制領域及能力，事情變得個容易分析。 即，如果 領域是限制的，足以允許它。

When a type system is an all-encompassing aspect of your program, we need to make sure we leave enough power in order not to overly constrain your expressiveness; without "type functions", you'd end up with quite a bit of boilerplate, e.g. hard-coded listOfInt, listOfString, listOfArrayOfFloat, their respective helper functions, etc. 然而，請也確保不會透過這個稍微強大的型別系統過度濫用這個能力。 有時候，寫些小型的 boilerplate 減少對其他額外強大型別的需求。 如果有區別的話，品味權衡突顯你的實用主義及判斷更甚於花俏的型別！
