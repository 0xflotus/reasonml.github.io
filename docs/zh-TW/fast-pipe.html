Fast Pipe

-> 是個能夠讓你「翻轉」你的程式碼的方便運算子。 a(b)<code> 能夠變成 <code>b->a 。它是一種沒有任何執行期間成本的語法。

假設您有以下內容:

validateAge(getAge(parseData(person)))

因為需要從最裡面讀到最外面，所以會有點難閱讀。使用 fast pipe 來簡化它。

person
  ->parseData
  ->getAge
  ->validateAge

基本上，parseData(person) 會被轉變為 person->parseData ，getAge(person->parseData) 會被轉變為 person->parseData->getAge，以此類推。

當函式擁有多個參數 (argument) 的時候依然有效。

a(one, two, three)

等同於

one->a(two, three)

這個對於標記的參數 (labeled argument) 也有效。

提示 & 技巧

請不要錯用 pipes。它們是被用來達成目的的。新手有時候會為了使用 pipe 的好處來改變函式庫的 API 。這樣是本末倒置的。

習慣上，我們不會把函式最裡面的部分放到 pipe 裡。所以上面的範例通常會被寫成：

parseData(person)
  ->getAge
  ->validateAge

JS 函式串接

閱讀以下的章節之前，需要先讀懂 Bucklescript's binding API。

JavaScript 的 API 通常會依附在物件 (object) 上，而且通常可以被串接，例如：

const result = [1, 2, 3].map(a => a + 1).filter(a => a % 2 === 0);

asyncRequest().setWaitDuration(4000).send();

假設我們不需要上面的串接行為，我們可以使用前一章節裡提到的 bs.send 來繫結 (bind) 每個情況。

[@bs.send] external map : (array('a), 'a => 'b) => array('b) = "";
[@bs.send] external filter : (array('a), 'a => 'b) => array('b) = "";

type request;
external asyncRequest: unit => request = "";
[@bs.send] external setWaitDuration: (request, int) => request = "";
[@bs.send] external send: request => unit = "";

你可以這樣使用它們：

let result = filter(map([|1, 2, 3|], a => a + 1), a => a mod 2 == 0);

send(setWaitDuration(asyncRequest(), 4000));

這比起 JS 相對來說看起來糟多了。 現在我們必須從最裡面讀到最外面來讀懂真正的邏輯。 我們也無法在這裡使用 |> 運算子，因為物件在繫結 (binding) 裡跑到最前面了。 不過 -> 卻仍然有效！

let result = [|1, 2, 3|]
  ->map(a => a + 1)
  ->filter(a => a mod 2 === 0);

asyncRequest()->setWaitDuration(400)->send;

Pipe 到 Variants

變異建構子，如 Some 或 Student，看起來像是函式，但它們不是，因為歷史因素。 有些時候，最好能夠當作函式使用。 Fast pipe 讓你可以這樣用！

let result = name->preprocess->Some

轉換成：

let result = Some(preprocess(name))

注意 把變異建構子當作函數在其他地方不會運作。

Pipe 佔位符

佔位符寫成底線告訴 Reason 你希望填入在後面的函數引數。下列兩個意義同等：

let addTo7 = (x) => add3(3, x, 4);
let addTo7 = add3(3, _, 4);

有些時候你不希望將值 pipe 到第一個位置。這種情況可以標注佔位符表示哪個你希望 pipe 進去的引數。

假設你有個函式 namePerson，接受 person 接著 name 引數。如果要轉換一個 person，則 pipe 會這樣寫：

makePerson(~age=47, ())
  ->namePerson("Jane");

如果有個 name 想要套用到 person 物件上，可以使用佔位符：

getName(input)
  ->namePerson(personDetails, _);

這讓你任何 pipe 到任意引數位置。同樣可以在具名引數使用：

getName(input)
  ->namePerson(~person=personDetails, ~name=_);
