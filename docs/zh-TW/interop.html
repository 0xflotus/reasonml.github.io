交互作用

直接在 Reason 撰寫 JavaScript

If you're just hacking things together, this can be very nice, but you also have all of the unsafety of JavaScript code

Js.log("this is reason");

[%bs.raw {| console.log('here is some javascript for you') |}];

{| 和 |} 是 OCaml 的多行字串分隔符號。 你也可以放上標籤，例如 {something| 接著會找尋相符的 |something} 作為結束。

以下為編譯後的 JavaScript：

// Generated by BUCKLESCRIPT VERSION 1.7.4, PLEASE EDIT WITH CARE
'use strict';
console.log("this is reason");
console.log('here is some javascript for you');

撰寫 JavaScript，讓 Reason 存取

如果希望在 Reason 取得 JavaScript 的值？

Js.log("this is reason");
let x = [%bs.raw {| 'here is a string from javascript' |}];
Js.log(x ++ " back in reason land"); /* ++ 是字串串接操作子 */

現在可能會想說"這是什麼巫術？？ OCaml 如何知道 x 是個字串？" 它不知道。 x 的型別是個魔術型別， 統一成任何東西！ 這有點危險而且對 OCaml 的型別推論演算法有級聯效應的影響。

let y = [%bs.raw {| 'something' |}];
Js.log(("a string" ++ y, 10 + y));
/* 危險！！OCaml 不會阻止你把 y 當作 2 種完全不同的型別 */

若要解決這個問題，你應該每次都為 bs.raw 提供具體型別。

let x: string = [%bs.raw {| 'well-typed' |}];
Js.log(x ++ " back in reason land");
/* 如果不把 x 當作字串使用，OCaml 會顯示錯誤 */

以下為輸出結果！

// Generated by BUCKLESCRIPT VERSION 1.7.4, PLEASE EDIT WITH CARE
'use strict';
console.log("this is reason");
var x = ( 'here is a string from javascript' );
console.log(x + " back in reason land");
var y = ( 'something' );
console.log(/* tuple */[
      "a string" + y,
      10 + y | 0
    ]);
var x$1 = ( 'well-typed' );
console.log(x$1 + " back in reason land");

前個章節的 2 個 %% 與這裡 1 個 % 的差異很重要！ [%%something ...] 是 OCaml "擴充點"代表最上層語句（例如，不可以出現在函式或值）。 [%something ...] 是個擴充點，充當表達式，可以放在任何地方 -- 但請確保放在裡面的 JavaScript 確實是個表達式！ 例如，不要在最後加上分號，若加上，當你執行編譯後 JavaScript 將會得到語法錯誤。

撰寫函式 & 傳遞值

我們需要一些認知，有關 BucketScript 執行期間各個值的代表。

  • strings 為 strings，ints 和 floats 只是 numbers
  • OCaml 的陣列是可變動、固定長度的清單，使用 javascript array 代表。
  • List 是不可變、函式風格的鏈表，也的確在多數情況下最慣用的一種。 However, its representation is more complicated (try Js.log([1,2,3,4]) to check it out). 因此，當與 JavaScript 溝通時，通常會轉成 & 轉自 Array，使用 Array.of_list 和 Array.to_list。
  • 如果想要深入瞭解，BuckleScript 文件有個清單。

了解之後，就可以使用 JavaScript 撰寫函式，接受 array 並傳回 number，沒有太多麻煩。

let jsCalculate: (array(int), int) => int = [%bs.raw
  {|
 function (numbers, scaleFactor) {
   var result = 0;
   numbers.forEach(number => {
     result += number;
   });
   return result * scaleFactor;
 }
|}
];

let calculate = (numbers, scaleFactor) => jsCalculate(Array.of_list(numbers), scaleFactor);

Js.log(calculate([1, 2, 3], 10)); /* -> 60 */

當然，上述函式可以無痛移植到 Reason。

請記住對於學習這是非常有用的逃生出口，這樣就可以快速跳入並且開始做點東西，但是回到開始並將東西轉成優良型別安全的 Reason 是很好的練習。

平靜下來且受到紀律

目前為止都在使用 bs.raw，非常快速和自由的方式，不適合正式環境。

但是如果我們確實需要呼叫 JavaScript 的函式呢？ 需要與 DOM 交互作用、或者使用 node 模組。 在 BuckleScript，使用 external 宣告式（文件）。

取得值和函式十分容易：

[@bs.val] external pi : float = "Math.PI";
let tau = pi *. 2.0;
[@bs.val] external alert : string => unit = "alert";
alert("哈嘍");

但是如果我們遇到更複雜？下列是如何呼叫 Canvas DOM node 的 getContext：

type canvas;

type context;

/* 這裡留下型別抽象，因為不會
 * 在其他地方直接使用 */
[@bs.send] external getContext : (canvas, string) => context = "";

let myCanvas: canvas = [%bs.raw {| document.getElementById("mycanvas") |}];

let ctx = getContext(myCanvas, "2d");

現在來分解發生什麼事。我們新增了抽象型別給 Canvas DOM node 及其附屬的 RenderingContext 物件。

接著製作 getContext 函式， 我們使用 @bs.send 而不是 @bs.val，接著給空字串。 @bs.send 表示"我們呼叫第一個參數的方法"，這個例子是 canvas。 給定上述，BuckleScript 將轉譯 getContext(theFirstArgument, theSecondArgument) 成 theFirstArgument.getContext(theSecondArgument, ...)。

空字串表示"JS的名稱就跟 BuckleScript 的 external 名稱相同" – 這個例子 getContext。 如果想要給予別的名稱（像是 getRenderingContext），那就必須提供 "getContext" 字串，這樣 BuckleScript 才會呼叫正確的函式。

加入更多有趣的函式。

[@bs.send] external fillRect : (context, float, float, float, float) => unit = "";

現在可以畫些東西！

fillRect(ctx, 0.0, 0.0, 100.0, 100.0);

這並不多，增加其他 canvas 方法很相似，接著可以做些真實有趣的東西。

編譯後的 JavaScript 看起來如何？

'use strict';
var tau = Math.PI * 2.0;
alert("hello");
var myCanvas = ( document.getElementById("mycanvas") );
var ctx = myCanvas.getContext("2d");
ctx.fillRect(0.0, 0.0, 100.0, 100.0);

Wow! 注意到 BuckleScript 直接未我們內聯（inline） pi ？結果看起來就像是手刻的程式碼。

使用現有的 JavaScript 函式庫

當人們為特定 JavaScript 函式庫撰寫繫結，通常會發佈到 npm。前往函式庫看如何找到這些繫結。

使用沒有綁定的函式庫，無論如何，你會希望如同往常先安裝 npm 套件，例如，使用 npm install --save <package-name>，接著繼續撰寫繫結。 你可能會發現 bs.module FFI 功能特別有幫助；根據 JS 編譯目標的模組格式，省略 import 或 require。

例如，這是 bs.glob 繫結程式碼（原始版本為 OCaml，已轉成 Reason）：

type error;

[@bs.module] external glob : (string, (Js.nullable(error), array(string)) => unit) => unit = "";

[@bs.val] [@bs.module "glob"] external sync : string => array(string) = "";

相關的 package.json 部份：

{
  "name": "bs-glob",
  "version": "0.1.0",
  ...
  "devDependencies": {
    "bs-platform": "^1.9.1"
  },
  "dependencies": {
    "glob": "^7.1.2"
  }
}
