List & Array

Список

Списки являются:

  • гомогенными
  • немутабельными
  • быстрыми для добавления элемента в начало
let myList = [1, 2, 3];

Reason lists are simple, singly linked lists.

Использование

Вы должны использовать список из-за возможности изменения размера, быстрого добавления в начало и быстрого разделения на несколько списков. При том, что все операции немутабельны!

Стандартная библиотека предоставляет модуль List (и его аналог спомеченными аргументами, ListLabels).

Немутабельное добавление в начало

Используйте spread синтаксис, который внутри List.cons:

let myList: list(int) = [1, 2, 3];
let anotherList = [0, ...myList];

myList не изменился. anotherList теперь [0, 1, 2, 3]. Это эффективное (константное время, не линейное). Три последних элемента anotherList разделены с myList!

Заметьте, что [a, ...b, ...c] синтаксически ошибочно. Мы не поддерживаем подобные операции для списка. Это была бы линейная операция (O(b)), так как каждый элемент b был бы по одному добавлен в c. Используйте List.concat для этого.

Изменение определенного элемента в середине списка не рекомендуется, так как затраты на производительность будут линейны (O(n)).

Доступ

switch (описанный всекции про паттер-матчинг) является обычным способом доступа к значениям списка:

let message =
  switch (myList) {
  | [] => "This list is empty"
  | [a, ...rest] => "The head of the list is the string " ++ string_of_int(a)
  };

Для доступа к конкретному элементу используйте List.nth.

Советы и трюки

Вы можете без проблем объявлять множество пустых списков. Как рассказано в секции про вариант для списка, пустой список — это конструктор варианта без аргумента, и компилируется в целое число. Так что нет дополнительных потерь.

To understand how prepend can be immutable and O(1) at the same time, see also the previous link.

Дизайн решения

In the future, we might provide an out-of-the-box list data structure that's immutable, resizable and features all-around fast operations.

Массив

Массивы как списки, кроме того, что они:

  • изменяемы
  • имеют быстрый произвольный доступ
  • фиксированного размера в нативном коде (в JS так же изменяемого размера)

Для определения используются [| и |].

let myArray = [|"hello", "world", "how are you"|];

Usage

Standard library Array and ArrayLabel module. For JS compilation, you also have the familiar Js.Array API.

Access & update an array item like so:

let myArray: array(string) = [|"hello", "world", "how are you"|];

let firstItem = myArray[0]; /* "hello" */

myArray[0] = "hey";

/* now [|"hey", "world", "how are you"|] */

Этот способ лишь синтаксический сахар для Array.get/Array.set.

Советы и трюки

Если вы компилируете код в JavaScript, то массивы Reason прямо ложатся на массивы JavaScript и наоборот. То есть даже учитывая, что массивы в нативном коде имеют фиксированный размер, вы все еще можете использовать Js.Array для их изменения. Это норма.
