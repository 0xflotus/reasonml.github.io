Глобальная установка

До настройки плагинов для редактора, вы должны установить нужные им глобальные бинарники.

(Рекомендованный способ) через Npm/Yarn

ПлатформаКоманда
macOSnpm install -g reason-cli@3.2.0-darwin
Linuxnpm install -g reason-cli@3.2.0-linux
WindowsPlease see https://github.com/reasonml/reasonml.github.io/issues/195

Примечание: npm не показывает статус загрузки, а сборки на данный момент тяжелые (~100 MB).

(Альтернативный способ) через OPAM

OPAM — это нативный менеджер пакетов для OCaml. Если вы пишли из OCaml разработки и у вас нет npm/yarn, то этот вариант годится. Только будьте осторожны!

Если у вас Windows, смотрите https://github.com/reasonml/reasonml.github.io/issues/195.

opam update
eval $(opam config env)
opam install reason.3.0.4 merlin.2.5.4

Устранение неполадок

Проблемы с установкой

Если установка провалилась, скорее всего вы используете npm 5.4.0 (npm --version). В npm версиях ниже 5.4.2 был баг, который уже исправлен. Обновите npm, и все должно заработать.

Если обновление не помогло, попробуйте вариант в https://github.com/reasonml/reasonml.github.io/pull/157. Если поможет, проголосуйте за него (просто мы не уверены, что это лучшее решение проблемы).

Если ничего не помогло, пишите нам в https://github.com/reasonml/reason-cli/issues.

Не работает плагин редактора

  • Если у вас Windows, инструментарий редактора под нее пока не очень отлажен. Помогите нам улучшить его: https://github.com/reasonml/reasonml.github.io/issues/195.
  • Попробуйте перезапустить свой редактор. Есть вероятность, что он просто не уловил обновления среды из оболочки shell (и, к примеру, не подгрузил таким образом новые бинарники).
  • Запустите следующую команду: ```readlink which ocamlmerlin refmt ocamlmerlin-reason```` На выходе вы должны получить три варианта пути. Если установкаreason-cliбыла успешной, каждый путь должен содержать в себе словоreason-cli` .
  • Проверьте версию Merlin: ocamlmerlin -version Команда должна вернуть "The Merlin toolkit version 2.5.x, for Ocaml 4.02.3". Не OCaml 4.03 или 4.04.

Сообщения ошибок: Unbound Module Js и прочие.

Убедитесь, что вы компилировали свой проект хотя бы один раз. Диагностика кода запустится только после этого. Команда для компиляции может отличаться в зависимости от проекта, но обычно это npm run build (которая в свою очередь вызывает bsb -make-world).

Если у вас Visual Studio Code, открывать редактор нужно из под корневой директории проекта (где находятся файлы package.json и bsconfig.json). Сделать это можно, к примеру, вызвав в терминале команду code . в корне проекта.
