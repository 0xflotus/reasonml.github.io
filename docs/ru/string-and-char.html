Строки и символы

Строка

Строки в Reason ограничиваются двойными кавычками (одинарные кавычки зарезервированы для символов).

let greeting = "Hello world!";
let multilineGreeting = "Hello
 world!";

Спецсимволы нужно экранировать:

let oneSlash = "\\";

Для конкатенации строк используется ++:

let greetings = "Hello " ++ "world!";

Quoted строка

Существует специальный синтаксис для строк, который позволяет

  • multiline string just like before
  • отсутствие необходимости в экранировании
  • хуки для специальных препроцессоров
let greetingAndOneSlash = {|Hello
World
\
Hehe...
|};

Это похоже на JS строки в обратных кавычках, но без необходимости экранирования и без интерполяции. Хотя вы можете легко сделать интерполяцию, как это сделал BuckleScript:

let world = {js|世界|js}; /* Supports Unicode characters */
let helloWorld = {j|你好，$world|j}; /* Supports Unicode and interpolation variables */

BuckleScript's special pre-processor can then look for such js and j markers around the string and transforms it into something else.

Использование

Большинствро строковых операций можно найти в стандартной библиотеке. For JS compilation, see the familiar Js.String API in the BuckleScript API docs. Так как строки Reason отображаются в строки JavaScript, вы можете смешивать и сопоставлять операции из обеих стандартных библиотек.

Советы и трюки

https://twitter.com/jusrin00/status/875238742621028355

Теперь у вас есть выразительная система типов! В нетипизированном языке вы часто излишне используете строки как:

  • уникальный идентификатор: var BLUE_COLOR = "blue"
  • идентификатор внутри структуры данных: var BLUE = "blue"; var RED = "red"; var colors = [BLUE, RED]
  • имя поля объекта: person["age"] = 24
  • перечисление: if (audio.canPlayType() === 'probably') {...} (ಠ_ಠ)
  • другие безумные паттерны, которые покажутся вам ужасными, после использования Reason.

The more you overload the poor string type, the less the type system (or a teammate) can help you! Reason дает краткие, быстрые и поддерживаем типы и структуры данных как альтернативу примерам выше (например, варианты, описанные в последующей секции).

В случае нативной компиляции, Reason строки компилируются в простое представление, производительность которого легко анализировать, но иногда требует ручной настройки. For example, naively concatenating strings like "hi " ++ "how " ++ "are " ++ "you?" unnecessarily allocates the intermediate strings "are you?" and "how are you?" (though it might be optimized into a single string in these simple cases). В этом случае лучше использовать String.concat. В некотором смысле, немного приятно, что традиционный анализ времени выполнения, который мы когда то изучали может быть полезен снова.

Under JavaScript compilation, a Reason string maps to a JavaScript string and vice-versa, so no such above concern or analysis opportunities apply.

Дизайн решения

Использование quoted строки позволяет делать интересные DSL. Такие как регулярные выражения:

let r = Str.regexp({|hello \([A-Za-z]+\)|});

в противоположность

let r = Str.regexp("hello \\([A-Za-z]+\\)");

Though for JS compilation, you'd use [%bs.re] and Js.Re instead, since Str is not available.

Акцент на простоту Reason/OCaml можно видеть в его прямолинейной реализации нативных строк. Слишком сложная реализация строки иногда может иметь обратный эффект.

Символ

Reason имеет тип данных в виде строки, содержащей только одну букву:

let firstLetterOfAlphabet = 'a';

Важно: Символы не поддерживают Unicode или UTF-8.

Советы и трюки

Символ компилируется в целое число от 0 до 255, для дополнительной скорости. Вы можете использовать паттерн-матчинг (описание далее) на нем:

let isVowel = (theChar) =>
  switch (theChar) {
  | 'a'| 'e'| 'i'| 'o'| 'u'| 'y' => true
  | _ => false
  };

To convert a String to a Char, use "a".[0]. To convert a Char to a String, use String.make(1, 'a').
